Q: Чем симметричное шифрование отличается от асимметричного?
A: 
**Symmetric Encryption**
	
- AES;
- один ключ для шифрования/расшифровки;
- быстро;
- проблема передачи ключа.
	
---
**Asymmetric Encryption**
	
- RSA;
- Public/Private Key;
- обмен ключами;
- медленнее симметричного.


Q: Расскажи о предназначениях каждого из механизмов криптографии.
A:   
	
| Mechanism         | Purpose                                    |
| ----------------- | ------------------------------------------ |
| Encoding          | Representation/compatibility               |
| Hashing           | Integrity, password storage                |
| Encryption        | Confidentiality                            |
| Digital Signature | Authenticity + Integrity + Non-repudiation |
	
- **Confidentiality** (конфиденциальность)
- **Integrity** (целостность)
- **Authenticity** (подлинность)
- **Non-repudiation** (неотказуемость)

Q: Почему не рекомендуется шифровать пароли для хранения?
A: это считается плохой практикой. Вместо шифрования используют **односторонние функции хэширования** (`bcrypt`, `Argon2`, `scrypt`).
	
Основная причина: **системе обычно не нужно знать исходный пароль пользователя**.
	
Когда пользователь входит в систему:
	
1. Пользователь вводит пароль.
2. Сервер вычисляет его хэш.
3. Сервер сравнивает полученный хэш с хэшем в базе.
	
Исходный пароль никогда не требуется восстановить.
