# OSI модель

**OSI** - Open Systems Interconnection


![[Pasted image 20260104182057.png]]

- Прикладной: [[HTTP 1.1 & HTTP 2 - Networks]]/HTTPS, FTP, SMTP, DNS, SNMP, SSH и др.
	
- Представления: SSL/[[TLS]], MIME, кодировки (ASCII, JPEG и др.).
    
- Сеансовый: [[RPC & RMI - Legacy Java|RPC]], NetBIOS sessions, PPTP.
    
- Транспортный: TCP, UDP, SCTP.
    
- Сетевой: IP, ICMP, ARP, OSPF, BGP.
	
- Канальный: Ethernet (IEEE 802.3), PPP, HDLC.
    
- Физический: стандарты передачи бит (Ethernet PHY, оптика, DSL, Wi‑Fi радиоинтерфейсы).


> [!note]
> сейчас RPC-фреймворки обычно относят к **прикладному уровню**, потому что они сами реализуют всё необходимое для поддержания состояния взаимодействия.
> 
> Например gRPC работает поверх HTTP/2 и TCP.

| Уровень OSI | Название                     | Основные задачи                                                                                      | Примеры протоколов/технологий                      |
| ----------- | ---------------------------- | ---------------------------------------------------------------------------------------------------- | -------------------------------------------------- |
| **7 (L7)**  | Прикладной (Application)     | Предоставляет сетевые сервисы приложениям пользователя                                               | HTTP, HTTPS, DNS, SMTP, FTP                        |
| **6 (L6)**  | Представления (Presentation) | Определяет формат данных: кодирование, сериализация, сжатие, шифрование                              | TLS, JSON, XML, Protobuf, JPEG                     |
| **5 (L5)**  | Сеансовый (Session)          | Устанавливает, поддерживает и завершает сеансы взаимодействия, управляет диалогом между приложениями | RPC, NetBIOS, SMB Session                          |
| **4 (L4)**  | Транспортный (Transport)     | Обеспечивает доставку данных между приложениями: сегментация, надёжность, контроль потока, порты     | TCP, UDP                                           |
| **3 (L3)**  | Сетевой (Network)            | Определяет маршрутизацию пакетов между сетями и логическую адресацию                                 | IP, ICMP, IPSec                                    |
| **2 (L2)**  | Канальный (Data Link)        | Передаёт кадры внутри одной сети, использует MAC-адреса, обнаруживает ошибки передачи                | Ethernet, Wi-Fi (802.11), PPP, MAC                 |
| **1 (L1)**  | Физический (Physical)        | Передаёт биты по физической среде: кабелям, радиоволнам, оптике                                      | Витая пара, оптоволокно, радиосигнал, Ethernet PHY |
