# UDP - Networks

UDP - User Datagram Protocol

UDP не устанавливает соединение и не гарантирует доставку данных.

При использовании UDP отправитель не проверяет доставку: здесь нет подтверждений. 

UDP также называют Fire-and-Forget protocol.



### UDP сохраняет границы сообщения

Предположим, приложение отправило UDP-датаграмму размером 4000 байт.

Если она не помещается в один IP-пакет, на сетевом уровне произойдёт **IP-фрагментация**:

```
UDP datagram (4000 bytes)
        ↓

IP fragment #1
IP fragment #2
IP fragment #3
```

На принимающей стороне IP сначала соберёт все фрагменты обратно в **одну UDP-датаграмму**, и только потом передаст её приложению.

Приложение всё равно увидит:

```
receive() -> 4000 bytes
```

а не три отдельных куска.

Если хотя бы один IP-фрагмент потеряется, обычно теряется вся UDP-датаграмма.

> каждому `send()` на стороне отправителя соответствует `receive()` на стороне получателя. Т.е. перепуток порядок байт в датаграмме невозможно.

> [!warning]
> UDP не поддерживает сегментацию -> приложения должны слать небольшие сообщения, которые могут вместиться в одну UDP datagram (верхнеуровневые протоколы отвечают за сегметнацию). 
>
> И вот доставка сегментов в том же порядке уже не гарантируется.
