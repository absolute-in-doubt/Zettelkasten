# Method Area - JMM

Область методов.

Общая для всех потоков.

### Что тут хранится

- **структуры для каждого класса:**
	- run-time constant pool - ~={cyan} для каждого класса свой,=~ _~={orange}общий для всех потоков=~_
	- данные `static` и `static final` полей и методов (~={cyan}таблица значений статических полей=~)
	- данные полей и методов обычных, -  ~={orange}БЕЗ ЗНАЧЕНИЙ, только метаданные=~(типы, модификаторы, атрибуты)
	- байткод для методов и конструкторов, включая **и статические, и обычные методы**, и методы интерфейсов.

> [!example] Note
> В Method Area хранится таблица статических полей, в которой хранится:
> - для примитива - **само значение примитива**
> - для ссылочного - **Java ссылка на объект**
>
>[[Run-time constant pool - JMM|Runtime constant pool]] при этом используется только как источник констант и символьных ссылок; когда класс инициализируется, из пула могут брать константное значение для `static final` и записывать его в соответствующий слот таблицы статических полей, но дальнейшее «живое» значение хранится именно в этом слоте, а не в constant pool.
>
>Разрешённые (interpreted) ссылки кэшируются в runtime constant poo или в метаданных класса, а не "сохраняются в таблице статических полей" — **~={orange}таблица статических полей хранит только значения=~** static переменных (primitive или ссылки на объекты в Heap).


Все потоки разделяют одну и туже область методов, поэтому ~={orange}доступ к данным из этой области =~и процесс динамического связывания~={orange} должен быть потоко‑безопасным=~. Если два потока пытаются получить доступ к полю или методу класса, который ещё не был загружен, то он должен быть загружен только один раз и оба метода не должны выполняться до тех пор, пока он не будет загружен.


> [!tip] Note
> Metaspace — реализация Method Area в HotSpot JVM.
>
>В HotSpot с Java 8 Metaspace заменил PermGen и размещается в нативной памяти, по умолчанию расширяясь динамически и ограничиваясь в основном доступной памятью ОС.


### Как это хранится

Метаданные в Method Area концептуально организованы **по классам**: для каждого загруженного класса есть своя «запись» с именем класса, родителем, интерфейсами, списком полей, списком методов, их сигнатурами и ссылками на байткод, а также его constant pool.
