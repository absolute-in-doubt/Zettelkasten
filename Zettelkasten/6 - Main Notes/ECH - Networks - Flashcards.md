Q: Расскажи как работает ECH
A:  ECH (Encrypted Client Hello) — это расширение TLS, которое шифрует содержимое ClientHello, включая SNI. Оно предотвращает раскрытие имени посещаемого сайта наблюдателям в сети и обычно используется совместно с защищённым DNS (DoH/DoT).
	
1. DNS-сервер публикует **ECH Config** (публичный ключ сервера для ECH).
    
2. Клиент получает этот ключ при DNS-запросе.
    
3. Клиент создаёт:
    
    - **Outer ClientHello** (видимый всем);
        
    - **Inner ClientHello** (с настоящим SNI).
        
4. `Inner ClientHello` шифруется публичным ключом из ECH Config.
    
5. Сервер расшифровывает внутренний `ClientHello` и продолжает TLS Handshake.
	
---
ECH обычно используется вместе с: 
	
```text
DNS over HTTPS (DoH)
или
DNS over TLS (DoT)
        +
ECH
```
	
Потому что обычный DNS-запрос тоже раскрывает доменное имя.

Q: Как работает DNS over HTTPS (DoH)?
A: DNS over HTTPS (DoH) — это способ отправки DNS-запросов внутри HTTPS-соединения. Клиент устанавливает TLS-соединение с DoH-сервером и передаёт DNS-сообщения как данные HTTP-запросов, благодаря чему DNS-запросы становятся зашифрованными и недоступными для наблюдателей в сети. 
	
Прикол в том, что при запросе к DNS мы тоже используем TLS, но адрес DNS нам не нужно скрывать. Поэтому не получается бесконечной loop.

Q: В чем смысл DOH + ECH, если ip адрес сервера сайта и так легко можно связать с доменом? 
A: **не всегда IP-адрес можно однозначно связать с доменом**. Именно поэтому комбинация DoH + ECH имеет смысл.
	
**Случай 1. Сайт имеет выделенный IP**
	
Если IP принадлежит только одному сайту:
	
```
203.0.113.10 -> only example.com
```
	
то да, провайдер может легко догадаться, что пользователь посещает `example.com`.
	
В этом случае ECH и DoH дают мало выигрыша.
	
---
**Случай 2. CDN и shared hosting (самый распространённый сегодня)**
	
Современный Интернет сильно опирается на CDN, такие как:
	
- Cloudflare
- Akamai Technologies
- Amazon Web Services
	
Один IP может обслуживать тысячи или даже миллионы доменов:
```
104.16.132.229
    ├── example.com
    ├── foo.com
    ├── myblog.net
    ├── company.org
    └── ...
```
В таком случае:
	
```
Client -> 104.16.132.229
```
	
не говорит наблюдателю, какой именно сайт посещается.
	
Именно для этого исторически и появился SNI: серверу нужно было понять, какой сертификат и какой сайт отдавать, когда на одном IP находятся тысячи доменов.
