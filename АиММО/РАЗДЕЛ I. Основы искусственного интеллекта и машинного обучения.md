## 1. Понятие и классификация ИИ

**Искусственный интеллект (ИИ)** — способность компьютерной системы выполнять задачи, требующие человекоподобного интеллекта: восприятие, рассуждение, обучение, принятие решений.

### 🗺️ Карта ИИ

![Карта типов ИИ](https://pplx-res.cloudinary.com/image/upload/pplx_search_images/268694039457447ee635084815f69a7e9a043d6c.jpg)

### Классификация по мощности (Capability)

```text
                   ИИ
         ┌─────────┴─────────┐
    Узкий (ANI)         Общий (AGI)      Сверхинтеллект (ASI)
  (1 задача)           (любая задача)    (превосходит человека)
  ✅ Существует         ❌ Не создан       ❌ Теория
  ChatGPT, AlexNet
```

### Классификация по функциональности (Russell & Norvig)

| Тип | Признак | Пример |
|-----|---------|--------|
| **Реактивные машины** | Нет памяти, только реакция | Deep Blue (шахматы) |
| **Ограниченная память** | Использует прошлые данные | Самоуправляемые авто |
| **Теория разума** | Понимает намерения других | Не создан |
| **Самосознание** | Осознаёт себя | Не создан |

> **Пример**: Google Translate — узкий ИИ с ограниченной памятью (контекст предложения).

### Вложенная структура: ИИ → МО → Глубокое обучение

![Вложенность AI, ML, DL](https://pplx-res.cloudinary.com/image/upload/pplx_search_images/ed7c44e8c27fac88235775b82963d60459d1a587.jpg)

```text
┌─────────────── ИИ ───────────────┐
│  ┌────────── МО ──────────┐      │
│  │  ┌─── Глубокое ─────┐  │      │
│  │  │   обучение (DL)  │  │      │
│  │  └──────────────────┘  │      │
│  └────────────────────────┘      │
└──────────────────────────────────┘
```

- **ИИ** — широкое понятие; включает любые системы, имитирующие интеллект.
- **МО** — подмножество ИИ; системы, которые **учатся на данных**.
- **DL** — подмножество МО; использует **многослойные нейронные сети**.

## 2. Понятие и классификация задач МО

**Машинное обучение (МО)** — раздел ИИ, где алгоритм улучшает свою производительность на задаче **T** по мере накопления опыта **E**, измеряемого метрикой **P** (определение Митчелла).

### Три парадигмы обучения

![Сравнение supervised, unsupervised и reinforcement learning](https://lh4.googleusercontent.com/ca39FFaiDeFrFMkhcHAl-J-XV4--sNmKKt1GoAQHeYnGTiHEsceOI6MMrWMPop13LK3uHAgV-UamNFjJx0kaOnCnVva1Q8oBvQBEfT4SXjXBuIg8L-UZFQWUrvxYYs2oMeZKUf5euIkWfTq9YNn6xDJleIwufHjmS8JZqO-GxRqW7i-RceyF0laAkg)

```text
                    ДАННЫЕ
                      │
         ┌────────────┼─────────────┐
         ▼            ▼             ▼
  С УЧИТЕЛЕМ    БЕЗ УЧИТЕЛЯ    С ПОДКРЕПЛЕНИЕМ
  (Supervised)  (Unsupervised)  (Reinforcement)
         │            │             │
   Есть метки    Меток нет      Есть награда
         │            │             │
  Классификация  Кластеризация   Агент→Действие
  Регрессия      Снижение       →Среда→Награда
                 размерности
```

**Обучение с учителем**
Обучение с учителем – это тип ML, при котором модель обучается на основе маркированных данных, где каждый пример данных имеет соответствующую целевую переменную или метку. Модель стремится найти зависимости и общие закономерности между входными данными и соответствующими выходными метками. Примеры алгоритмов обучения с учителем включают линейную регрессию, метод опорных векторов, случайные леса и нейронные сети.

Примеры задач:

Классификация – определение принадлежности объекта к конкретному классу. Например, классификация электронных писем на спам.
Регрессия – прогнозирование непрерывной целевой переменной. Например, предсказание цены недвижимости на основе ее характеристик.


**Обучение без учителя**
Обучение без учителя – это тип ML, при котором модель обучается на немаркированных данных без явной целевой переменной. Вместо этого, модель ищет скрытые структуры, паттерны и группы в данных. Алгоритмы обучения без учителя используются для кластеризации данных, снижения размерности, ассоциативного анализа и генерации новых признаков.

Примеры задач:

Кластеризация – группировка схожих объектов внутри данных. Например, сегментация покупателей на основе их покупательского поведения.
Снижение размерности – уменьшение размерности данных, сохраняя важные признаки и устраняя шум. Например, сжатие изображений без значительной потери информации.


**Обучение с подкреплением**
Обучение с подкреплением – это тип ML, в котором модель обучается взаимодействуя со средой и получая обратную связь в виде награды или штрафа. Модель самостоятельно принимает решения и настраивает свое поведение на основе полученной награды. Его активно используют в робототехнике, играх и управлении автономными системами.

Примеры задач:

Управление роботом – обучение робота совершать определенные действия в окружающей среде для достижения поставленных целей.
Игры – обучение агента играть в игры, например, шахматы или видеоигры, чтобы достичь наивысшего возможного счета.

#### Обучение с учителем — разбор на примере

```text
Входные данные (X)         Метки (Y)
площадь квартиры: 80 м²  → цена: $100k    ← Регрессия
email текст              → спам/не спам   ← Классификация
```

**Задачи:**

- **Классификация** — предсказать категорию (спам / не спам, кошка / собака).
- **Регрессия** — предсказать непрерывное число (цена, температура).

#### Обучение без учителя — разбор на примере

```text
Входные данные (X)         Метки (Y)
покупки клиентов          → ??? (нет меток)
                          ↓ алгоритм сам
                    Кластер A: молодёжь
                    Кластер B: пенсионеры
```

**Задачи:** кластеризация, снижение размерности (PCA), поиск аномалий.

#### Обучение с подкреплением — разбор на примере

```text
Агент (бот)  ──действие──▶  Среда (игра)
     ▲                           │
     └──────── награда ◀─────────┘
              (+1 за победу, -1 за поражение)
```

> **Пример**: AlphaGo — агент учился играть в Го, получая +1 за победу.

### Полная карта задач МО

| Парадигма | Задача | Алгоритмы | Пример |
|-----------|--------|-----------|--------|
| Supervised | Классификация | SVM, Decision Tree, KNN | Спам-фильтр |
| Supervised | Регрессия | Linear Reg., XGBoost | Прогноз цены |
| Unsupervised | Кластеризация | K-means, DBSCAN | Сегментация клиентов |
| Unsupervised | Снижение размерности | PCA, t-SNE | Визуализация |
| Reinforcement | Оптимизация политики | Q-learning, PPO | Игровые агенты |

## 4. Типы закономерностей при анализе данных

**Закономерность** — устойчивая зависимость в данных, которую модель должна обнаружить.

```text
            ТИПЫ ЗАКОНОМЕРНОСТЕЙ
                    │
    ┌───────────────┼────────────────┐
    ▼               ▼                ▼
Ассоциации      Кластеры         Тренды
(A → B)        (группы схожих)   (во времени)
    │               │                │
Пример:         Пример:          Пример:
"Памперсы +     Клиенты A, B, C  Рост продаж
 пиво" →        покупают похоже  каждый декабрь
 правила
 ассоциаций
```

| Тип | Описание | Метод обнаружения |
|-----|---------|------------------|
| **Ассоциации** | Совместное появление объектов | Apriori, FP-Growth |
| **Классификация** | Принадлежность к категории | Decision Tree, SVM |
| **Кластеризация** | Схожие группы без меток | K-means, DBSCAN |
| **Регрессия** | Непрерывная зависимость | Linear/Polynomial Reg. |
| **Тренды** | Изменение во времени | ARIMA, LSTM |
| **Аномалии** | Отклонения от нормы | Isolation Forest |

> **Пример ассоциации**: Amazon — «с этим товаром часто покупают...» — правило вида `{диван} → {подушки}` с поддержкой 40%, достоверностью 70%.

## 5. Обучающая, валидационная и тестовая выборки

### Разбиение датасета — схема

```text
                    ВЕСЬ ДАТАСЕТ (100%)
    ┌────────────────────────────────────────┐
    │         Train (70–80%)                 │   ← учимся
    ├────────────────────┬───────────────────┤
    │   Validation (10–15%)                  │   ← подбираем гиперпараметры
    ├────────────────────┴───────────────────┤
    │         Test (10–20%)                  │   ← итоговая оценка (1 раз!)
    └────────────────────────────────────────┘
```

**Обучающая выборка (Train set)** — данные, на которых модель **подбирает веса/параметры**.

**Валидационная выборка (Validation set)** — данные для **подбора гиперпараметров** (глубина дерева, learning rate). Модель её «видит» косвенно.

**Тестовая выборка (Test set)** — данные, которые модель **не видела никогда**. Оценка на ней — честная оценка обобщения.

> ⚠️ **Утечка данных (Data Leakage)**: если тестовые данные использовались при обучении — результат завышен и недостоверен.

### Cross-Validation (K-fold) — когда данных мало

```text
K=5:
Fold 1: [TEST][TRAIN][TRAIN][TRAIN][TRAIN]
Fold 2: [TRAIN][TEST][TRAIN][TRAIN][TRAIN]
Fold 3: [TRAIN][TRAIN][TEST][TRAIN][TRAIN]
Fold 4: [TRAIN][TRAIN][TRAIN][TEST][TRAIN]
Fold 5: [TRAIN][TRAIN][TRAIN][TRAIN][TEST]
                     ▼
          Итоговая метрика = среднее по 5 фолдам
```

## 6. Проблема переобучения: Bias–Variance Tradeoff

### Центральная диаграмма: кривая ошибки

![Bias-Variance tradeoff](https://www.researchgate.net/publication/335604816/figure/fig2/AS:799391489220609@1567601191401/Bias-variance-trade-off-in-machine-learning-This-figure-illustrates-the-trade-off.png)

```text
 Ошибка
   │          Общая ошибка = Bias² + Variance + Irreducible Error
   │      ╲
   │       ╲         ╱ (Variance растёт с усложнением)
   │        ╲       ╱
   │         ╲     ╱
   │          ╲   ╱
   │    Bias   ╲ ╱  ← оптимальная точка
   │ (убывает)  ╳
   │            │
   └────────────────────────────────────► Сложность модели
         ^           ^
     Underfitting  Overfitting
```

### Определения

**Смещение (Bias)** — систематическая ошибка: модель слишком проста → не улавливает структуру данных.

**Дисперсия (Variance)** — чувствительность к шуму: модель слишком сложна → запоминает шум обучающих данных.

**Irreducible Error** — неустранимый шум в данных; не зависит от модели.

### Мишень (Bullseye) — визуальная аналогия

![Мишень (Bullseye) — визуальная аналогия](https://encrypted-tbn0.gstatic.com/images?q=tbn:ANd9GcTLX3rlEgtxv86VxL95omHy-VklQnexPVHSNg&s)

```text
           Low Variance    High Variance
              ●●●              ·  · ·
High Bias     ●●●●   vs.      · ·   ·
              ●●●               ·
           (стабильно,       (разброс,
           но смещено)       но вразброс)

              ●                 · · ·
Low Bias     ● ● ●   vs.       ·  · ·
              ●               ·   ·  ·
           (идеал!)          (нестабильно)
```

### Сравнение: Underfitting vs. Overfitting

| | Underfitting | Good Fit | Overfitting |
|--|--|--|--|
| **Bias** | Высокий | Низкий | Низкий |
| **Variance** | Низкая | Низкая | Высокая |
| **Train error** | Высокая | Низкая | Очень низкая |
| **Test error** | Высокая | Низкая | **Высокая** |
| **Пример модели** | Линия для нелин. данных | Оптимальная глубина дерева | Дерево без ограничений |

### Диаграмма: три кривые на данных

![Диаграмма: три кривые на данных](https://miro.medium.com/v2/resize:fit:1400/1*BA0RHYcvs8ml_JIDkIzUzQ.jpeg)

### Формула декомпозиции ошибки

\[
\text{Error}(x) = \text{Bias}^2 + \text{Variance} + \sigma^2
\]

\[
\text{Bias}^2 = \left(\mathbb{E}[\hat{f}(x)] - f(x)\right)^2
\]

\[
\text{Variance} = \mathbb{E}\left[\left(\hat{f}(x) - \mathbb{E}[\hat{f}(x)]\right)^2\right]
\]

### Методы борьбы с переобучением

```text
Переобучение (High Variance)?
├── Добавить данных (Data Augmentation)
├── Регуляризация (L1, L2, Dropout)
├── Уменьшить сложность модели
├── Early Stopping
└── Cross-Validation

Недообучение (High Bias)?
├── Усложнить модель
├── Добавить признаки (Feature Engineering)
├── Снизить регуляризацию
└── Обучать дольше
```
