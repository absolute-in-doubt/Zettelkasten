Если кратко пробежаться по проектам:
1. Сервис истории операций - сервис хранящий историю бизнес операций. Получал их из Kafka. Было два набора эндпоинтов: user-facing + admin endpoints. 
	
   Лично я занимался написанием API:
	- внедрил пагинацию + фильтрацию
	- внедрил кэширование
	- оптимизировал БД - сидели с DBA анализировали логи и slow transaction alerts -> добавили индексы -> ускорил медленные транзакции в 10 раз (500мс -> 50мс)
	- писал unit тесты, в том числе Testcontainers, для тестирования Kafka consumer.
	
   Стек:
   Java 17, Spring Boot 3.1, PostgreSQL, JUnit 5. (Spring Kafka, Testcontainers)
---
2. Сервис сторис - мкиросервис для публикации коротких сторис для пользователей сайта и приложения. Сторис создаются и управляются менеджерами банка через админ панель без участия тех персонала.
	
   Команда: 
	- TeamLead - тоже писал код + 2 backend включая меня.
	- 2 frontend
	- 2 Android
	- 2 IOS
	- 2 тестировщика
	- 1 аналитик
	- 1 PM
	
   **Ключевые характеристики:**
	•     Единый контент для всех пользователей — сторис не персонализированы
	•     Управление только вручную менеджерами через Admin Panel (без технического персонала)
	•     Поддержка веб-приложения и мобильного приложения через единый API
	•     Мультислайдовые сторис с медиа (изображение / видео) и CTA-кнопкой
	•     Планирование публикации и автоматическое снятие по сроку
	•     Базовая аналитика: просмотры и клики по CTA
	
- **Спроектировал и реализовал Redis-кэш для публичного API стори**с, снизив p99-латентность с ~450 мс до 80 мс и обеспечив устойчивость к пиковой нагрузке 1 000+ RPS при нулевой дополнительной нагрузке на БД.
	
- **Реализовал Transactional Outbox** для гарантированной exactly-once доставки событий в Kafka — устранил 100% риска потери события при сбое сервиса и обеспечил 0 ms дополнительной задержки на критическом пути запроса.
	
- **Спроектировал pipeline загрузки медиа** (4 формата, до 50 МБ) с S3-интеграцией, **magic-byte валидацией**, **асинхронной генерацией превью** и pre-signed URL — сократил Time-to-Publish для менеджеров с часов до <2 мин.
	
- **Реализовал событийную интеграцию (Avro + Schema Registry) между сервисом сторис и сервисом уведомлений** — обеспечил автоматическую доставку push-уведомлений 100% подписанных пользователей в течение <5 сек после публикации.
	
  Cтек: Java 17. Spring Boot 3.1, Kafka 3, Redis, JUnit 5

3. Сервис уведомлений - сервис принимающий из Kafka события для отправки уведомлений. Отправлял 4 типа уведомлений: push, sms, email, website notifications. 
   Реализовано было всё, кроме website notifications, через сторонних провайдеров. 
   На самом сервисе у нас была только inbox таблица, таблица для хранения уведомлений сайта, таблица шаблонов уведомлений и таблица user notification preferences.
	
Моя роль в проекте:
- реализовал inbox pattern + оттестил с Testcontainer -> получилось добиться exactly once гарантии.
- Настроил Redis кэширование для шаблонов уведомлений и user notification preferences.
- Настроил OpenTelemetry (Spring Actuator)
- Написал Admin API
	
  Стек: Java 17, Spring Boot 3.1, Kafka 3, Redis, JUnit 5, Testcontainers
