## 1. «Грязные данные»: источники и классификация

**Грязные данные** — данные, содержащие ошибки, несоответствия или пропуски, снижающие качество модели.


### Классификация проблем

| Тип | Пример | Причина |
|-----|--------|---------|
| **Пропуски (Missing values)** | `age = NaN` | Не заполнено, сбой датчика |
| **Дубликаты** | Одна строка записана дважды | Ошибка слияния баз |
| **Аномалии (Outliers)** | `age = 999` | Ошибка ввода |
| **Несогласованность** | `"москва"` / `"Москва"` / `"MOSCOW"` | Разные источники |
| **Некорректный тип** | Дата как строка `"01-01-2020"` | Нет схемы данных |
| **Шум** | Случайные флуктуации в сенсорных данных | Погрешность измерения |


### Источники грязных данных

```
Источники грязных данных
        │
┌───────┼────────┬──────────────┐
▼       ▼        ▼              ▼
Ручной  Интегра- Устаревшие    Отсутствие
ввод    ция из   системы       стандартов
        разных   хранения      сбора
        систем
```

> **Правило "Garbage in — Garbage out"**: модель обучится воспроизводить ошибки из данных, а не находить закономерности.

## 2. Очистка данных: пропуски и аномалии

### 2.1 Устранение пропусков

**Пропуск (Missing value)** — отсутствующее значение в признаке объекта.

#### Три типа пропусков (по механизму)

| Тип | Расшифровка | Что делать |
|-----|-------------|------------|
| **MCAR** (Missing Completely at Random) | Пропуск случаен, не зависит от данных | Удалить строки |
| **MAR** (Missing at Random) | Зависит от других признаков | Импутация |
| **MNAR** (Missing Not at Random) | Зависит от самого пропущенного значения | Отдельная категория |

#### Стратегии устранения пропусков

```
Пропуск в признаке X
        │
┌───────┼───────────┬──────────────┐
▼       ▼           ▼              ▼
Удалить  Заполнить  Заполнить      Модель
строку   Mean/      соседним       предсказывает
(если    Median/    значением      значение
<5%)     Mode       (ffill/bfill)  (KNN, MICE)
```

> **Пример**: в датасете о ценах жилья пропуски в `garage_size` — не случайны: дома без гаража просто не имеют значения. Создаём признак `has_garage = 0`.


### 2.2 Обнаружение и устранение аномалий

**Аномалия (Outlier)** — наблюдение, значительно отклоняющееся от общего распределения данных.

#### Метод IQR (межквартильный размах)

$$
IQR = Q_3 - Q_1
$$

Граница выброса:
$$
\text{нижняя} = Q_1 - 1.5 \cdot IQR \quad;\quad \text{верхняя} = Q_3 + 1.5 \cdot IQR
$$

![IQR](https://loginom.ru/sites/default/files/blogpost-files/data_general-post_outliers-article_image_box-plot.svg)

#### Методы обнаружения аномалий

| Метод | Принцип | Когда |
|-------|---------|-------|
| **IQR / Boxplot** | Статистический порог | Одномерный случай |
| **Z-score** | \|z\| > 3 → выброс | Нормальное распределение |
| **Isolation Forest** | Случайная изоляция редких точек | Многомерный случай |
| **LOF** | Плотность соседей | Пространственные данные |

#### Что делать с выбросом?

```
Выброс найден
      │
┌─────┼──────┬──────────┐
▼     ▼      ▼          ▼
Удалить  Заменить  Clipping   Оставить
(ошибка  на       (cap to     (реальный
 ввода)  median   boundary)   феномен)
```

## 3. Нормализация и стандартизация

**Масштабирование** — приведение признаков к сопоставимому диапазону, чтобы алгоритм не отдавал предпочтение признакам с большими числами.

> **Пример**: `возраст` (0–100) и `зарплата` (1000–100000) — без масштабирования зарплата доминирует в метриках расстояния.


### 3.1 Min-Max нормализация

**Min-Max нормализация** — линейное преобразование значений в диапазон $$[0, 1]$$.

$$
x' = \frac{x - x_{\min}}{x_{\max} - x_{\min}}
$$

```
До:  [10, 20, 30, 40, 50]
       ↓ Min=10, Max=50
После: [0.0, 0.25, 0.5, 0.75, 1.0]
```

⚠️ Чувствителен к выбросам: один экстремум растягивает весь диапазон.

### 3.2 Z-score стандартизация (StandardScaler)

**Z-score стандартизация** — преобразование данных к нулевому среднему и единичной дисперсии.

$$
z = \frac{x - \mu}{\sigma}
$$

где $$\mu$$ — среднее, $$\sigma$$ — стандартное отклонение.

```
До:  [10, 20, 30, 40, 50]  μ=30, σ=14.14
       ↓
После: [-1.41, -0.71, 0, 0.71, 1.41]
```

✅ Подходит, когда данные близки к нормальному распределению.

### 3.3 Robust Scaling

**Robust Scaling** — масштабирование через медиану и IQR; устойчив к выбросам.

$$
x' = \frac{x - \text{median}(x)}{IQR}
$$

### Когда что применять

| Метод | Диапазон | Чувствителен к выбросам | Когда использовать |
|-------|----------|------------------------|-------------------|
| **Min-Max** | [1] | ⚠️ Да | CNN, нейросети; данные без выбросов |
| **Z-score** | (-∞, +∞) | ⚠️ Умеренно | SVM, PCA, линейные модели |
| **Robust** | (-∞, +∞) | ✅ Нет | Данные с выбросами |

## 4. Преобразование данных

### 4.1 Кодирование категориальных переменных

**Категориальная переменная** — признак, принимающий значения из ограниченного множества категорий (например: `"красный"`, `"синий"`, `"зелёный"`).

[

#### Label Encoding — порядковое кодирование

Присваивает каждой категории целое число.

```
Цвет:   красный → 0
        синий   → 1
        зелёный → 2
```

⚠️ Создаёт искусственный порядок: модель может решить, что `зелёный > красный`.
✅ Подходит только для **порядковых** (ordinal) переменных: `плохо < нормально < хорошо`.

#### One-Hot Encoding

Создаёт бинарный столбец для каждой категории.

```
До:                После:
Цвет              красный  синий  зелёный
красный    →         1       0       0
синий      →         0       1       0
зелёный    →         0       0       1
```

✅ Подходит для **номинальных** переменных (без порядка).
⚠️ При большом числе категорий — **проблема высокой размерности**.


#### Target / Frequency Encoding

- **Target Encoding**: заменяет категорию средним целевого признака по этой категории.
- **Frequency Encoding**: заменяет категорию частотой её встречаемости.

```
Город     → Target Encoding (средняя цена)
Москва    → 150 000
Минск     → 85 000
```

### 4.2 Бинаризация

**Бинаризация** — преобразование непрерывного признака в 0/1 по порогу.

$$
x' = \begin{cases} 1 & \text{если } x \geq \theta \\ 0 & \text{если } x < \theta \end{cases}
$$

> **Пример**: возраст → `is_adult`: если `age ≥ 18` → 1, иначе → 0.

### 4.3 Полиномиальные признаки

**Полиномиальные признаки** — новые признаки, порождённые степенями и попарными произведениями исходных.

Для признаков $$x_1, x_2$$ при степени $$d=2$$:

$$
[x_1, x_2] \rightarrow [1,\; x_1,\; x_2,\; x_1^2,\; x_1 x_2,\; x_2^2]
$$


```
Зачем: линейная модель не может описать нелинейную зависимость.
       Полиномиальные признаки делают её нелинейной без смены алгоритма.

x:  [1, 2, 3]
       ↓ degree=2
x²: [1, 4, 9]   ← новый признак
```

⚠️ При высоких степенях — риск переобучения.

## 5. Аугментация данных для нейронных сетей

**Аугментация данных** — искусственное увеличение объёма обучающей выборки за счёт генерации модифицированных копий существующих данных.

**Цель**: снизить переобучение, улучшить обобщающую способность модели.

### 5.1 Аугментация изображений

#### Геометрические трансформации

```
Оригинал → Поворот (rotate) → Отражение (flip) → Обрезка (crop)
         → Масштабирование (zoom) → Сдвиг (shift)
```

#### Фотометрические трансформации

```
Оригинал → Яркость (brightness) → Контраст (contrast)
         → Насыщенность (saturation) → Шум (noise)
```

| Техника | Что делает | Пример |
|---------|-----------|--------|
| **Flip** | Зеркальное отражение | Кот смотрит влево/вправо — одно и то же |
| **Rotate** | Поворот на ±15°–30° | Цифра `6` под наклоном всё равно `6` |
| **Crop** | Случайная вырезка | Лицо в разных частях кадра |
| **Color Jitter** | Изменение цвета | Та же кошка в тёмной комнате |
| **Cutout/Dropout** | Случайное закрашивание | Модель учится по части объекта |
| **Mixup** | Смешение двух изображений | Плавный переход между классами |

### 5.2 Аугментация текста

```
Оригинал: "Кот сидел на диване"
    ↓ Синонимы
"Котик находился на кушетке"
    ↓ Перефразирование (backtranslation)
EN: "The cat sat on the sofa" → RU: "Кот сидел на диване"
```

### 5.3 Аугментация табличных данных

- **SMOTE** — генерирует синтетические примеры миноритарного класса путём интерполяции между соседями.
- **Gaussian Noise** — добавление малого шума к числовым признакам.

$$
x'_i = x_i + \varepsilon,\quad \varepsilon \sim \mathcal{N}(0, \sigma^2)
$$

> **Пример**: датасет болезней с 95% здоровых и 5% больных — SMOTE дополняет класс «больных» синтетическими примерами.

***

## Итоговая шпаргалка: выбор метода

```
Данные → Проверить пропуски
              ↓
         > 5%? → Импутация (KNN, Median)
         < 5%? → Удалить строки
              ↓
         Найти выбросы (IQR / Z-score)
              ↓
         Ошибка? → Удалить
         Реальный? → Robust Scaler или оставить
              ↓
         Числовые признаки → Min-Max / Z-score
         Категории → One-Hot (nominal) / Label (ordinal)
              ↓
         Нейросеть? → Аугментация
         Нелинейность? → Полиномиальные признаки
```
