Q: В чем разница сервисно-ориентированной и микросервисной архитектуры?
A:   
**Сервисно-ориентированная архитектура**
	
- сервисы - минимальные жизнеспособные утилиты, которые могут быть переиспользованы. Сервисы создаются ради сервисов, чтобы другие их использовали. Конечной идеи - самому собрать их этих серсиов большое приложение - нет.
- **Цель:** возможность переиспользовать сервисы.
- **Пример:** ip-lookup service - сервис, который вызывают различные приложения, чтобы проерить геопозицию пользователя. Он может быть использован в разных приложениях разных направлений.
	- По сути Keycloak тоже пример такого сервиса
	
**Микросервисная архитектура**
	
- сервисы - bounded contexts. Есть конечное понимание приложения, собранного из разрабатываемых сервисов. You're okay with the thought that this is probably the only use of this microservice.
- **Цель:** - горизонтальное масштабирование bottleneck-ов
