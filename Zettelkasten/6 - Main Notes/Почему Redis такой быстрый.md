# Почему Redis такой быстрый



### 1. Хранение данных в RAM

![[Pasted image 20260507185654.png]]

Processor Reegister; L1, L2 и L3 Caches - мы не можем контролировать.

RAM - самая быстрая память, которую мы можем контролировать.

### Однопоточность

![[Pasted image 20260507185930.png]]

`+` Не нужно никаких блокировок.

Так как всё в оперативе ()чтение быстрое, нет стопящих IO-bound операций) - один поток справляется, а внедрение многопоточности только замедлило бы работу, т.к. потребовались бы блокировки и синхронизация.
