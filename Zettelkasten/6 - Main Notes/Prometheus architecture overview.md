# Prometheus architecture



![[prometheus-architecture.gif]]


### 🔥Prometheus Server

Отвечает за 
- сбор данных из различных источников
- хранение их в Time-Series DB.
- выполнение запросов данных через HTTP API.

**Components:**
- **Retrieval** (входная воронка) - собирает данные с различных эндпоинтов. Находит эндпоинты для сбора инфы через:
	  - static configuration
	  - dynamic service discovery
- **Time-Series DB** 
- **HTTP Server** - предоставляет API для 
	- запрашивания данных из TIme-Series DB, используя PromQL
	- взаимодействия с другими компонентами экосистемы Prometheus

С чего конкретно собирает данные в Kubernetes cluster:
 
- `pod/monitoring-prometheus-node-exporter-b8thd` - node exporter - собирает матрики о состоянии ноды (будь то worker node or a master node)

- `pod/monitoring-kube-state-metrics-f56d7b58b-mncws` - kube state metrics - собирает метрики о всём кластере

- `/metrics` scraper - собирает данные о метриках приложений, развёрнутых в кластере


---
### 📤 Pushgateway


- The Pushgateway is used to expose metrics from short-lived jobs or applications that cannot be scraped directly by Prometheus.
- These jobs push their metrics to the Pushgateway, which then makes them available for Prometheus to scrape(pull).
- Use Case:
    - It's particularly useful for batch jobs or tasks that have a limited lifespan and would otherwise not have their metrics collected.


---
### 🚨 Alertmanager


- The Alertmanager is responsible for managing alerts generated by the Prometheus server.
- It takes care of deduplicating, grouping, and routing alerts to the appropriate notification channels such as PagerDuty, email, or Slack.


---
### 🧲 Exporters

Exporters - маленькие приложения, которые собирают метрики из различных сторонних систем и предоставляют их в формате, который Prometheus может интерпретировать.

 They are essential for monitoring systems that do not natively support Prometheus.
 
- Types of Exporters:
    - Common exporters include the Node Exporter (for hardware metrics), the MySQL Exporter (for database metrics), and various other application-specific exporters.


**Примеры в Kubernetes cluster:**
- `pod/monitoring-prometheus-node-exporter-b8thd`
- `pod/monitoring-kube-state-metrics-f56d7b58b-mncws`
