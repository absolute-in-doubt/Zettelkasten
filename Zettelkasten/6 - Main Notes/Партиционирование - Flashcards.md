
Theory for the cards: [[Партиционирование]]

FILE TAGS: system_design databases

Q: Для чего нужно партиционирование?
A: Когда индекс по таблице становится очень большим, при поиске возникают проблемы с производительностью. 
	
После разделения на разделы (партиции), у каждой таблицы будет свой индекс, поэтому поиск в таблицах меньшего размера будет выполняется быстрее.

Q: Какие типы шардирования ты знаешь?
A:    
**Горизонтальное** (по строкам): 
	
таблица разбивается на подмножества строк по ключу (например, по ID или дате), каждая партиция хранит все столбцы.
	
Пример: 
- отделяем данные в разные партиции по дате. Например в Outbox одна неделя -> одна партиция, потом на некст неделю новая партиция, а вторую фул дропаем.
- храним пользователей разных континентов в своей партиции, так, что европеец ищет в шарде европы. И эти шарды разнести по континентам в реплики БД.
	
	
**Вертикальное** (по столбцам): 
	
таблица делится на несколько таблиц по столбцам, часто группируя часто используемые вместе; подходит для "широких" таблиц.
	
Часть чаще используемых столбцов - в одну партицию, те, что реже - в другую.
	
	
**Функциональное**: 
	
данные группируются по доменам или контексту использования (например, данные о пользователях на одном сервере, о заказах — на другом).
