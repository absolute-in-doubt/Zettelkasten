## 1. Принцип ансамблирования и Bias–Variance

**Ансамбль** — это композиция нескольких базовых моделей, объединённых в одно решение.  
Идея: «много слабых мнений» в сумме дают более устойчивый и точный прогноз.

![Схема ансамбля: несколько моделей → голосование](https://habrastorage.org/getpro/habr/upload_files/653/3b0/6ef/6533b06efe605995702ebec3b174bb45.png)  
Источник: [Habr — Ансамблевое обучение для самых маленьких](https://habr.com/ru/articles/814981/)

**Bias–Variance декомпозиция** объясняет, почему ансамбли помогают.

- **Bias (смещение)** — систематическая ошибка модели (слишком простая, недообучение).  
- **Variance (разброс)** — чувствительность к конкретной выборке (переобучение).  

Графическая интуиция (мишень):

- Высокий bias: предсказания кучно, но далеко от центра.  
- Высокий variance: предсказания разбросаны, но вокруг центра.  
- Хорошая модель: низкий bias и умеренный variance.

Ансамбли:

- **Бэггинг** снижает variance (усреднение независимых моделей).  
- **Бустинг** снижает bias, постепенно исправляя ошибки предыдущих моделей.

## 2. Bootstrap и бэггинг (Bagging)

### 2.1 Bootstrap

**Bootstrap‑выборка** — это выборка с возвращением из исходного датасета.  
Размер bootstrap‑выборки обычно равен размеру исходной.

Пример: было 100 объектов → случайно берём 100 с возвращением.  
Часть объектов повторится, часть не попадёт (out-of-bag).

Схема из proproprogs:

- Детально: [Статья — «Случайные деревья и случайный лес. Бутстрэп и бэггинг»](https://proproprogs.ru/ml/ml-sluchaynye-derevya-i-sluchaynyy-les-bootstrap-i-bagging){web:342}

### 2.2 Принцип бэггинга

**Bagging (Bootstrap Aggregating)** — ансамбль, обучающий одни и те же модели на разных bootstrap‑подвыборках и усредняющий ответы.

Пайплайн:

1. Сгенерировать B bootstrap‑выборок из обучающего набора.  
2. На каждой выборке обучить базовый алгоритм (часто дерево решений).  
3. Для регрессии: среднее предсказаний.  
4. Для классификации: голосование по большинству.

Интуитивная картинка (bagging vs boosting):

![Bagging vs Boosting (параллельно vs последовательно)](https://cdn.otus.ru/media/public/43/11/1_zTgGBTQIMlASWm5QuS2UpA_1-1801-431178.jpeg)  
Источник: OTUS — ансамбли моделей[image:357]

Плюсы бэггинга:

- Снижает разброс (variance).  
- Хорошо работает с «шумными» базовыми моделями (глубокие деревья).

## 3. Случайный лес (Random Forest)
![](https://proproprogs.ru/htm/ml/files/ml-sluchaynye-derevya-i-sluchaynyy-les-bootstrap-i-bagging.files/image001.jpg)

### 3.1 Идея случайного леса

**Random Forest** — ансамбль из многих деревьев решений, обученных по схеме бэггинга + случайный подвыбор признаков.

Ключевые «случайности»:

- Bootstrap по объектам.  
- Случайный набор признаков при выборе лучшего сплита в каждом узле (feature bagging).

Схема идеи: деревья обучаются на разных подвыборках и видят разные подмножества признаков.

```text
Датасет → bootstrap #1 → Дерево 1
        → bootstrap #2 → Дерево 2
        ...
        → bootstrap #B → Дерево B
Предсказание = голосование / среднее
```

Детальная статья с рисунками:  
[https://proproprogs.ru/ml/ml-sluchaynye-derevya-i-sluchaynyy-les-bootstrap-i-bagging]

### 3.2 Алгоритм Random Forest (кратко)

1. Выбрать число деревьев \(B\).  
2. Для каждого дерева:  
   - bootstrap‑выборка объектов;  
   - на каждом сплите выбирать лучший разрез только по случайному подмножеству признаков (mtry).  
3. Аггрегировать предсказания деревьев (голосование / среднее).

### 3.3 Важность признаков

Важность признака в случайном лесе часто измеряют как:

- Суммарное снижение impurity (Gini / entropy) по всем сплитам, где используется признак.  
- Или падение качества при его случайной перестановке (permutation importance).

Пример с важностями признаков легко найти в примерах proproprogs по случайному лесу.

## 4. Стекинг (Stacking)

![alt](https://habrastorage.org/r/w1560/getpro/habr/upload_files/669/1be/795/6691be7955cdd52269ee31704d7d82c0.jpg)

**Stacking** — ансамбль, где выходы нескольких разных моделей подаются на вход «метамодели» (второго уровня).

Пайплайн:

1. Обучить разные базовые модели: kNN, дерево, логистическая регрессия, SVM и т.п.  
2. Для валидационных объектов записать их предсказания как новые признаки.  
3. На этих новых признаках обучить метамодель (часто логистическую регрессию или другой простой алгоритм).  
4. На тесте: сначала предсказания базовых моделей → затем метамодель.

Схема:

```text
Х → Модель 1 → p1 ┐
   Модель 2 → p2 ├→ Метамодель → итоговый прогноз
   Модель 3 → p3 ┘
```

Интуиция:

- Каждый базовый алгоритм видит данные «под своим углом».  
- Метамодель учится комбинировать их сильные стороны.

Статья с обзором способов сборки ансамблей (в том числе стекинг):  
<https://habr.com/ru/articles/561732/>

## 5. Градиентный бустинг

### 5.1 Принцип бустинга

**Boosting** — последовательное обучение слабых моделей, где каждая следующая исправляет ошибки предыдущих.

Схема на картинке (Habr):

![Последовательное добавление деревьев в бустинге](https://habrastorage.org/getpro/habr/upload_files/9ca/63d/c6a/9ca63dc6a0f1144ea611b7547c140a1f.png)  
Источник: [Habr — Ансамблевое обучение для самых маленьких][https://habr.com/ru/articles/814981/](image:358)

Отличие от бэггинга:

- Бэггинг — модели независимы, обучаются параллельно.  
- Бустинг — модели зависят, обучаются последовательно.

### 5.2 Псевдоостатки в градиентном бустинге

Градиентный бустинг обобщает AdaBoost на произвольные функции потерь.

Пусть есть модель \(F_m(x)\) после \(m\) шагов.  
Следующее слабое дерево строится по **псевдоостаткам** — антиградиенту функции потерь по предсказаниям.

Общая схема:

1. Инициируем модель \(F_0(x)\) константой (например, средним по \(y\)).  
2. На шаге \(m\) считаем псевдоостатки \(r_{im} \approx -\partial L(y_i, F(x_i))/\partial F(x_i)\).  
3. Обучаем слабый регрессор \(h_m(x)\) предсказывать эти остатки.  
4. Обновляем модель: \(F_m(x) = F_{m-1}(x) + \nu \cdot h_m(x)\), где \(\nu\) — learning rate.  
5. Повторяем до нужного числа итераций.

Статья с формулами и графиками:  
<https://proproprogs.ru/ml/ml-gradientnyy-boosting-i-stohasticheskiy-gradientnyy-boosting>

### 5.3 XGBoost, LightGBM, CatBoost (очень кратко)

Все они — реализации градиентного бустинга над деревьями, но с разными инженерными решениями.

- **XGBoost** — оптимизации по скорости, регуляризации, обработке спарс‑данных.  
- **LightGBM** — градиентный бустинг с подходом leaf-wise и эффективной работой с большими данными.  
- **CatBoost** — бустинг с автоматическим обработчиком категориальных признаков и техникой устранения таргет‑ликейджа.

В разделе proproprogs много примеров реализации градиентного бустинга на Python.

## 6. Преимущества и недостатки ансамблевых методов

### 6.1 Преимущества

- **Высокая точность**: ансамбли часто побеждают одиночные модели.  
- **Снижение переобучения** (у бэггинга, RF).  
- **Устойчивость к шуму и выбросам** (усреднение).  
- Возможность сочетать разные типы моделей (стекинг).

Для случайного леса:

- Не требует тонкой настройки многих параметров.  
- Хорошие out-of-the-box результаты во многих задачах.

Для градиентного бустинга:

- Очень гибкий, позволяет использовать произвольные функции потерь.  
- Высокое качество на табличных данных (XGBoost / CatBoost — стандарт индустрии).

### 6.2 Недостатки

- **Сложность интерпретации**: отдельные деревья интерпретируемы, ансамбль из сотен — уже нет.  
- **Время обучения и предсказания** выше, чем у одиночной модели.  
- Градиентный бустинг **чувствителен к переобучению**, требует ранней остановки, регуляризации и настройки learning rate.  
- Стекинг требует аккуратной кросс‑валидации, иначе легко получить утечку информации.

## 7. Быстрая карта раздела

```text
Ансамбли
 ├─ Что даёт ансамблирование?
 │   ├─ Снижение variance (бэггинг, Random Forest)
 │   └─ Снижение bias (бустинг)
 ├─ Bootstrap
 │   └─ Бэггинг
 │       └─ Random Forest (случайные объекты + случайные признаки)
 ├─ Стэкинг
 │   └─ Модели первого уровня → метамодель
 └─ Градиентный бустинг
     ├─ AdaBoost (веса объектов)
     ├─ Gradient Boosting (псевдоостатки)
     └─ XGBoost / LightGBM / CatBoost
```
