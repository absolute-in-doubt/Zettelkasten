Q: Что за аннотация @Enumerated? Для чего она нужна? Какие параметры принимает?
A: @Enumerated - аннотация для Enum поля Entity. Определяет то, как это поле будет конвертироваться для хранения в БД.
	
Аргументы:
- `EnumType.ORDINAL` - int представление этого Enum. Индексирует начиная с 0.
	**Плюсы**: Компактно. 
	**Минусы**: Изменение порядка/удаление enum ломает данные.
	
- `EnumType.STRING` - получает `toString()` данного Enum insance и хранит в БД в виде String.

Q: Какие есть ограничения и ттребования к Entity классам в JPA?
A:     
**Основные требования**
	
- **Аннотация `@Entity`**: Класс обязательно помечен `@Entity` или зарегистрирован в XML-конфигурации (orm.xml).​
    
- **Конструктор без аргументов**: Должен быть `public` или `protected` (не `private`). Другие конструкторы разрешены.
	
- **Первичный ключ**: Обязательно наличие `@Id` (одно поле или составной через `@IdClass`/`@EmbeddedId`).
    
- **Top-level класс**: Только обычный класс верхнего уровня, не вложенный, не анонимный.​
    
	
**Ограничения**
	
- **Не final класс**: Класс не может быть `final` (провайдеру нужен прокси для lazy loading)
    
- **Не enum/интерфейс**: Запрещены `enum`, интерфейсы, аннотации.​
    
- **Не final методы/поля**: Persistent поля и методы с маппингом (`@Id`, `@Column` и т.д.) не могут быть `final`.
    
- **Serializable для remote**: Если передается по сети (detached), должен реализовывать `Serializable`.​
    
	
**Доступ к полям**
	
Поддерживается field access (аннотации на полях) или property access (на геттерах). Поля не должны быть публичными для внешнего доступа — используйте геттеры/сеттеры
