# Хранение данных - Redis


![[Pasted image 20260507183542.png]]

По сути Redis - один большой dictionary.


### Иерархия хранимых данных

В Redis есть возможность выстраивать иерархию с помощью ключей. Например:
```
user:2564:name
```

- `user` - во главе иерархии
	- `2564` - потомок `user`
		- `name` - потомок `2564`


---
### Keys


![[Pasted image 20260507184103.png]]

`SomeKey`, `someKey` и `somekey` - будут считаться разными ключами


---
### Values

![[Pasted image 20260507184356.png]]

Redis автоматически распознаёт тип вставляемых данных.

- При вставке "100500" распознает, что это целое число и сохранит как int, таким образом, сэкономив место.

- В Redis нет типа данных для дробных чисел -> сохранит "100.5" как строку.



---
### Как хранить данные?

![[Pasted image 20260507191641.png]]

- Легко интегрировать - имеется в виду "easy to implement".

Стандартные библиотеки не предоставляют готовых хороших сериализаторов для Hashes/RedisJson. Если искать сторонние реализации - там зачастую бывают утечки памяти + ненадежно. 

В Ozon используют хранение в виде json. И изменяют объект напрямую: `GET` - достали, обновили, `SET` - положили обратно.
