# Многоверсионность и Конкурентность - PostgreSQL


MVCC - MultiVersion Concurrency Control


#### Упрощённо, про так как работает MVCC

![[Pasted image 20260208194028.png]]

При любом UPDATE, DELETE мы строки из базы не удаляем. Мы порождаем новую версию записи


![[Pasted image 20260208194113.png]]

При удалении запись помечается (крестиком), давая понять, что запись уже не актуальна. Но сама запись из таблицы не исчезает.


---
### Как это работает на самом деле

В реале, в Postgres нет колонки Version. Контроль за доступностью записи производится посредством снимков данных.

![[Pasted image 20260208194807.png]]


- `txmin` - номер транхакции, начиная с которой, эта запись видна.
- `txmax` - номер транзакции, начиная с которой, эта запись не видна.
