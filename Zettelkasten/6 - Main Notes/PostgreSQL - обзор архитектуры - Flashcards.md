Q: Кратко расскажи об архитектуре Postgres.
A:    
![[Pasted image 20260325123053.png|697]]
	
Зелёное - кэши различных процессов
	
**Фоновые процессы:**
	
- **Archiver** - позволяет архивировать логи
- **Auto Vacuum** - очистка устаревших логов, обновление карт видимости и свободного пространства, обновление статистики
- **WAL** - журнал, куда записываются все изменения, которые должныпроизойти по результату транзакций.  **Выполняем действия в оперативной памяти** - отдаём пользователю ответ, раньше, чем он будет записан на диск. WAL log скидывает все поступающие запросы на диск.
- **Checkpointer** - создаёт "чекпоинты": начиная с данного лога WAL, все остальные уже были обработаны - данные уже на диске -> можно удалить эти записи из WAL log.
- **Writer** - переносит данные на диск (асинхронно)
- **Stats Collector** - заведует обработкой статистики
- **Logging Collector** - отвечает за сбор логов.
	
---
**Взаимодействие с клиентом**
	
> [!tip] Note
> Каждое соединение с Postgres client обрабатывается отдельным backend. -> используем Pooler для поддержания несокльких соединений.
	
	
- **Pooler** держид соединение со всем backend процессами - не закрывает и не пересоздаёт
