# RPC & RMI - Legacy Java


### RPC

![[Pasted image 20260210103716.png]]

`+` Универсальная штука (Использует HTTP/1, TCP/IP) - может использоваться в различных языках программирования, таких как C, C++, Python и т.д.

`-` RPC не поддерживает ООП: нельзя передавать объекты в RPC методы

- марашллизирует (XML) передаваемые данные


> [!note] 
> RPC - скорее описание стандарта,, который может быть реализован по разному (например [[gRPC|gRPC]], в то время как RMI - чисто реализация.

---
### RMI

`java.rmi`

`-` Java-local история: использует JRMP - Java Remote Method Protocol

`+` поддерживает ООП: можно передавать объекты в RMI методы


- сериализует (последовательность байтов) передаваемые данные

![[Pasted image 20260210112210.png]]

Ссылки на удалённые объекты (хранящиеся на сервере) хранятся в rmiregistry

1. Распределенное приложение RMI использует регистратор (_registry_) для получения ссылки на удаленный объект. 
2. Сервер вызывает регистратор для связывания имени с удаленным объектом. 
3. Клиент ищет удаленный объект по его имени в регистраторе сервера, а затем вызывает на нем метод
