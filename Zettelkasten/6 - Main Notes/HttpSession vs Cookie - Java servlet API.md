# HttpSession vs Cookie - Java servlet API

| Характеристика    | `HttpSession`                                                                     | `Cookie`                                                               |
| ----------------- | --------------------------------------------------------------------------------- | ---------------------------------------------------------------------- |
| **Где хранится**  | ~={orange}На сервере=~ (данные сессии лежат в памяти/репозитории сервера)         | На клиенте (браузер хранит cookie и отправляет их в каждом запросе)    |
| **Что хранит**    | Объекты Java (любые данные, связанные с пользователем)                            | Только строки (ключ‑значение)                                          |
| **Идентификация** | Клиент получает `JSESSIONID` cookie, чтобы сервер знал, к какой сессии обращаться | Cookie сами по себе могут хранить любые данные, включая идентификаторы |
| **Безопасность**  | Данные не видны клиенту, только идентификатор                                     | Данные видны клиенту, можно изменить вручную                           |
| **Время жизни**   | Управляется сервером (можно задать таймаут сессии)                                | Управляется браузером (через `Max-Age`, `Expires`)                     |
| **Использование** | Для хранения состояния пользователя: корзина, авторизация, временные данные       | Для хранения настроек клиента: язык, тема, токен авторизации           |
