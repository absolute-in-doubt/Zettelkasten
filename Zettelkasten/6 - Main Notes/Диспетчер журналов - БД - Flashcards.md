
Theory for the cards: [[Диспетчер журналов (WAL) - PostgreSQL]]

FILE TAGS: postgres databases

Q: Что такое WAL log и для чего он нужен?
A:     
1. Параллельно:
	- обработка запроса (выполнение) в RAM
	- запись в диспетчер журналов
2. Сохранение записи диспетчера журналов на диск
3. ~={orange}Возврат результата выполнения команды пользователю=~
4. Сохранение результатов выполнения команды на диск (потихонечку асинхронно)
	
`+` при падении БД выполнив записи из WAL log можно вернуть БД в консистентное состояние
