# Платформа управления тарифами и комиссиями карточных продуктов

### Архитектура

Внутренняя система управления тарифами и комиссиями по дебетовым и кредитным картам банка.

Система использовалась бизнес-подразделениями для настройки условий обслуживания карточных продуктов без необходимости внесения изменений в код прикладных систем.

Архитектура представляла собой модульный монолит с выделенными интеграционными модулями.

Интеграции:

- Core Banking System (получение информации о договорах и карточных продуктах)
    
- Карточный процессинг
    
- CRM банка
    
- Интернет-банк
    
- Мобильный банк
    
- Система выпуска банковских карт
    
- Хранилище нормативно-справочной информации
    

Основные сценарии:

- настройка комиссий за переводы;
    
- настройка комиссий за снятие наличных;
    
- настройка стоимости обслуживания карт;
    
- настройка лимитов операций;
    
- версионирование тарифов;
    
- публикация изменений тарифов во внешние системы.

- Spring Data JPA
    
- PostgreSQL
    
- Liquibase
    
- REST API
    
- OpenAPI
    
- Maven
    
- Git
    

Мониторинг:

- Spring Actuator
    
- Prometheus
    
- Grafana
    

### Что делал

- Разрабатывал REST API для создания, изменения и публикации тарифов карточных продуктов.
    
- Реализовал механизм версионирования тарифов с возможностью отложенного вступления изменений в силу.
    
- Участвовал в проектировании модели данных для хранения тарифов, комиссий и лимитов.
    
- Реализовал бизнес-логику расчета применимого тарифа с учетом типа карты, сегмента клиента и даты действия тарифного плана.
    
- Разрабатывал интеграции с Core Banking для получения информации о договорах и продуктовых пакетах клиентов.
    
- Участвовал в реализации синхронизации тарифных изменений с карточным процессингом.
    
- Писал SQL-запросы для формирования отчетов по изменениям тарифов и анализа использования карточных продуктов.
    
- Оптимизировал медленные запросы, связанные с поиском активных тарифных конфигураций.
    
- Разрабатывал миграции Liquibase и сопровождал изменения схемы БД.
    
- Покрывал бизнес-логику unit-тестами и участвовал в code review.
    

### Достижения

- Снизил p95 latency API получения тарифных параметров с 420 мс до 180 мс за счет оптимизации SQL-запросов и добавления составных индексов.
    
- Сократил время публикации новых тарифов с нескольких часов до 15–20 минут благодаря автоматизации процесса согласования и применения конфигураций.
    
- Уменьшил количество инцидентов, связанных с рассинхронизацией тарифов между карточным процессингом и внутренними системами, более чем на 50%.
    
- Снизил время выполнения наиболее ресурсоемких SQL-запросов с 1.5–2 секунд до 250–300 мс после рефакторинга запросов и пересмотра индексной стратегии.
    
- Разработал механизм аудита изменений тарифов, позволивший сократить время расследования инцидентов и анализа изменений конфигурации.
