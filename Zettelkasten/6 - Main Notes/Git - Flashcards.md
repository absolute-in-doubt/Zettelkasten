Q: Как удалить определённый файл из индекса?
A: `git reset <path_to_file>`
> [!warning] **Не путать с `git reset --soft/--hard <commit>`**
> `git reset --soft <commit>` - удалит коммиты после указанного, но оставит все файлы, которые изменились после этого коммита
> 
> `git reset --hard <commit>` - удалит все коммиты до указанного, а также все изменения

Q: Чем` git reset <file>` отличается от `git rm <file>`?
A: - **`git reset <file>`** (по умолчанию `--mixed`): Удаляет файл из staging area (index), **оставляя изменения в рабочей директории**. Файл остаётся на диске, незатронутым, но не подготовленным к коммиту.
    
- **`git rm <file>`**: Удаляет файл **и из рабочей директории, и из индекса**. Файл исчезает с диска, а в Git добавляется запись об удалении для следующего коммита.
