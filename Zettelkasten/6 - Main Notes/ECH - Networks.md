# ECH - Networks


**ECH (Encrypted Client Hello)** — расширение TLS, предназначенное для сокрытия содержимого `ClientHello`, прежде всего поля **SNI (Server Name Indication)**.

Ранее эта технология называлась **ESNI (Encrypted SNI)**, но затем была расширена до защиты практически всего `ClientHello`.

---

### Зачем нужен ECH?

Без ECH провайдер или любой наблюдатель может увидеть:

```text
ClientHello
    SNI = www.example.com
```

Даже если дальнейший трафик шифруется через HTTPS.

С ECH:

```text
Outer ClientHello
    SNI = публичный фронтенд/заглушка

Encrypted ClientHello
    SNI = www.example.com
```

Реальное имя сайта становится недоступным для наблюдателя.

---

### Что именно скрывается?

Шифруется почти весь внутренний `ClientHello`, включая:

- настоящий `SNI`;
    
- список поддерживаемых расширений;
    
- некоторые параметры TLS.
    

Не скрываются:

- IP-адрес сервера;
    
- факт использования TLS;
    
- размер и время передачи пакетов.
    

---

### Как работает?

1. DNS-сервер публикует **ECH Config** (публичный ключ сервера для ECH).
    
2. Клиент получает этот ключ при DNS-запросе.
    
3. Клиент создаёт:
    
    - **Outer ClientHello** (видимый всем);
        
    - **Inner ClientHello** (с настоящим SNI).
        
4. `Inner ClientHello` шифруется публичным ключом из ECH Config.
    
5. Сервер расшифровывает внутренний `ClientHello` и продолжает TLS Handshake.
    

---

### Какие технологии обычно используются вместе?

```text
DNS over HTTPS (DoH)
или
DNS over TLS (DoT)
        +
ECH
```

Потому что обычный DNS-запрос тоже раскрывает доменное имя.

---

### Что видит провайдер?

Без ECH:

```text
IP: 142.x.x.x
SNI: www.example.com
```

С ECH:

```text
IP: 142.x.x.x
TLS используется
```

Если несколько сайтов разделяют один IP (CDN), определить конкретный сайт становится значительно сложнее.

---

### Ограничения

- Работает только при поддержке **клиентом, DNS и сервером**.
    
- Не скрывает IP-адрес сервера.
    
- Не обеспечивает анонимность (для этого обычно используют VPN или сеть вроде The Tor Project).
    

---

### Краткий ответ для собеседования

> ECH (Encrypted Client Hello) — это расширение TLS, которое шифрует содержимое ClientHello, включая SNI. Оно предотвращает раскрытие имени посещаемого сайта наблюдателям в сети и обычно используется совместно с защищённым DNS (DoH/DoT).
