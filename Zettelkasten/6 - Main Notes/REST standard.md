# REST standard


![[Pasted image 20260208214611.png]]




REST (Representational State Transfer) — это архитектурный стиль для построения сетевых API, основанный на принципах HTTP и работе с ресурсами через стандартные методы. 

---

## Краткий конспект по REST

- ~={cyan}**Ресурсы и URL**  =~
    Любой объект (пользователь, товар, заказ) рассматривается как _ресурс_. Ресурс доступен по уникальному URL, например `/users/123`.
    
- ~={cyan}**HTTP‑методы и CRUD**  =~
    REST‑сервисы обычно сопоставляют методы HTTP с операциями CRUD:
    
    - `GET` — чтение ресурса. **~={orange}idempotent=~**
        
    - `POST` — создание ресурса. ~={red}**not idempotent**=~
        
    - `PUT` — полное обновление ресурса. **~={orange}idempotent=~**
        
    - `DELETE` — удаление ресурса. **~={orange}idempotent=~**
        
    - `PATCH` — частичное обновление ресурса.
		
	- `HEAD` — запрос разрешенных заголовков (для [[CORS|CORS]]) **~={orange}idempotent=~**
        
- ~={cyan}**Без состояния (stateless)**  =~
    Каждый запрос от клиента к серверу должен содержать всю необходимую информацию. Сервер не хранит состояние сессии между запросами, что упрощает кэширование и масштабирование.practicum.yandex+1
    
- ~={cyan}**Единообразный интерфейс**  =~
    API использует единую схему URL, стандартные статус‑коды (200, 201, 404, 400 и т.п.) и форматы ответов (чаще всего JSON).

## Какие HTTP‑методы должны быть идемпотентными по REST

Идемпотентный метод — это такой, при котором повторный одинаковый запрос не меняет состояние сервера сильнее, чем первый. То есть нельзя «удалить дважды» или «обновить больше, чем один раз» в смысле логики данных.developer.mozilla+1

По стандарту идемпотентными считаются:

- `GET` — получение данных, не меняет состояние сервера.
    
- `HEAD` — то же, что `GET`, но без тела ответа.
    
- `PUT` — полное обновление/создание ресурса по указанному URL; повторный одинаковый `PUT` даёт один и тот же результат.
    
- `DELETE` — удаление ресурса; повторный `DELETE` на удалённый ресурс обычно возвращает 404, но сам ресурс уже удалён.
    

Не является идемпотентным:

- `POST` — предназначен для создания ресурса или выполнения действия с побочными эффектами; каждый вызов может создавать новый объект или изменять состояние иначе.habr+1
    

Метод `PATCH` формально может быть идиоматически идемпотентным или нет в зависимости от реализации, поэтому в «чистом» REST‑подходе обычно считают идемпотентными только `GET`, `HEAD`, `PUT` и `DELETE`.babok-school+1

### Пример REST API

| HTTP метод | Пример URL         | Назначение                                             |
| ---------- | ------------------ | ------------------------------------------------------ |
| POST       | `/users`           | Создать нового пользователя                            |
| GET        | `/users`           | Получить список пользователей                          |
| GET        | `/users/new`       | возвращает HTML форму (страницу)  создания записи      |
| GET        | `/users/{id}/edit` | возвращает HTML форму (страницу) редактирования записи |
| GET        | `/users/{id}`      | Получить пользователя по ID                            |
| PUT/PATCH  | `/users/{id}`      | Обновить данные пользователя                           |
| DELETE     | `/users/{id}`      | Удалить пользователя                                   |
