# Helm vs kubectl - Helm Chart


Helm chart — это пакетный менеджер для Kubernetes, представляющий собой набор шаблонов YAML-файлов, которые упрощают развертывание, управление и версионирование приложений в кластере. Он включает метаданные (Chart.yaml), значения по умолчанию (values.yaml) и шаблоны ресурсов (templates/), позволяя параметризировать конфигурации для разных окружений.kurshub+1

## Основные компоненты

- **Chart.yaml**: содержит имя, версию, описание и зависимости чарта.​
    
- **values.yaml**: файл с параметрами, которые подставляются в шаблоны для генерации манифестов Kubernetes.​
    
- **templates/**: директория с шаблонами Deployment, Service и других ресурсов, использующими шаблонизацию Helm (Go-шаблоны).​
    

Чарты хранятся в репозиториях, поддерживают зависимости и позволяют легко делиться приложениями.[​

## Отличия от kubectl

Kubectl — это CLI-инструмент для прямого взаимодействия с Kubernetes API: apply, get, delete и т.д. для отдельных манифестов.​

|Аспект|Helm chart|kubectl|
|---|---|---|
|**Уровень**|Пакеты приложений с шаблонами|Отдельные команды для ресурсов|
|**Шаблонизация**|Поддерживает параметры и values.yaml|Нет, требует ручных правок YAML|
|**Управление**|Установка, upgrade, rollback релизов|Ручное применение манифестов|
|**Зависимости**|Автоматическое разрешение|Ручное создание зависимых ресурсов|
|**Версионирование**|Встроенное для чартов и релизов|Нет, зависит от Git или скриптов kurshub+1|

Helm использует kubectl под капотом, но добавляет абстракцию для сложных приложений, снижая дублирование YAML и ошибки.​
