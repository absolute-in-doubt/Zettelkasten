# C4 diagram


C4 - четыре 'C' - Context, Containers, Components, Code - стандарт для изображения структуры микросервисных приложений.

Проектирование идёт по декомпозиции: сначала Context, затем его декомпозиция - Container level, затем - Component level, затем - Code.

### 1. Context

Само приложение представлено в виде одного крупного блока ("черный ящик"), показывающего её взаимодействие с внешними системами и пользователями. 

![[Pasted image 20260202173617.png]]




### 2. Container

Показывает разделение системы на контейнеры: веб-серверы, приложения и базы данных.

Backend и Frontend - всё еще черные ящики, пусть backend и состоит из несольких контейнеров.

![[Pasted image 20260202173947.png]]


Тут на схеме контейнеров видно, что клиентское приложение (Frontend) отправляет к серверному (Backend) запросы и команды. Это соответствует шаблону **_разделения ответственности запросов и команд (CQRS,_** **_Command_** **_Query_** **_Responsibility_** **_Segregation)_**, где _командами_ называются изменения (мутации) данных, т.е. операции их создания, изменения и удаления, а _запросы_ означают выполнение операций чтения, что не изменяет данные.


### 3. Component

Отображает внутреннее устройство выбранного контейнера (в нашем случае это _Backend)_, разделенное на компоненты.
Компоненты представляют собой самостоятельные части системы, которые выполняют определенные функции, т.е микросервисы и то, что необходимо для их взаимодействия.
