# Happens before guarantee - Java Multithreading



Happens-before — это отношение «порядка» между действиями **~={red}в разных потоках=~**, гарантируемое Java Memory Model (JMM).
- Если A happens-before B, то:
  - Все эффекты записи, сделанные до/в A, становятся видимыми потоку, выполняющему B.
  - B не может «увидеть» состояние памяти так, будто A не произошло.

Важно: happens-before — логический порядок видимости, а не обязательно реальный порядок выполнения на процессоре.


---
### Visibility

Visibility — что реально может прочитать поток, когда другие потоки уже что-то записали.

Каждый поток может кэшировать значения переменных в регистрах/локальных кешах, не обращаясь к общей памяти.

Без синхронизации возможны:

  - Видимость старых значений (stale data): один поток обновил переменную, другой продолжает видеть старое значение.
  
  - Перестановки (reordering): компилятор и CPU могут менять порядок инструкций, если это безопасно внутри одного потока, но без синхронизации это ломает межпоточный порядок.

Следствие: «простой» флаг boolean running = true; в одном потоке и цикл while (running) в другом без синхронизации может никогда не завершиться.


> [!example] **Happens before + volatile**
> volatile запрещает некоторые оптимизации компилятора и процессора, обеспечивая:
>  1. Visibility:
>     - Запись в volatile-переменную публикует (flush) изменения в общую память.
>     - Чтение volatile-переменной обновляет значение из общей памяти, а не из кеша.
>  2. Happens-before:
>     - Запись в volatile-переменную happens-before последующего чтения этой же переменной другим потоком.
>     - Это значит: все записи в обычные переменные, произошедшие до записи в volatile, становятся видимыми потоку, который позже прочитал volatile.
