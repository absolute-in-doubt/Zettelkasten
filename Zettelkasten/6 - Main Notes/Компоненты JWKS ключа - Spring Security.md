# Компоненты JWKS ключа - Spring Security


### **1. `kty` (Key Type)**

- **Назначение:** Определяет тип ключа (алгоритм).
- **Возможные значения:**
    - `RSA` — для RSA-ключей (асимметричные).
    - `EC` — для эллиптических кривых (например, `P-256`, `P-384`).
    - `oct` — симметричный ключ (например, для HMAC).
    - `OKP` — для ключей на основе кривых Эдвардса (например, Ed256519).
- **Формат хранения:** Строка (`"RSA"`, `"EC"` и т.д.).

---

### **2. `kid` (Key ID)**

- **Назначение:** Уникальный идентификатор ключа в наборе (JWKS).
- **Использование:** Помогает клиентам выбрать нужный ключ для валидации JWT (например, в заголовке JWT указывается `kid`).
- **Формат хранения:** Строка (например, `"key-1"`, `"2024-05-29-01"`).

---

### **3. `alg` (Algorithm)**

- **Назначение:** Алгоритм, для которого предназначен ключ.
- **Возможные значения:**
    - `RS256`, `RS384`, `RS512` — для RSA + SHA-256/384/512.
    - `ES256`, `ES384`, `ES512` — для ECDSA + SHA-256/384/512.
    - `HS256`, `HS384`, `HS512` — для HMAC + SHA-256/384/512.
    - `EdDSA` — для Ed25519.
- **Формат хранения:** Строка (`"RS256"` и т.д.).

---

### **4. `use` (Public Key Use)**

- **Назначение:** Указывает, для чего используется ключ.
- **Возможные значения:**
    - `sig` — для подписи (signature).
    - `enc` — для шифрования (encryption).
- **Формат хранения:** Строка (`"sig"`, `"enc"`).

---

### **5. Параметры ключа (зависят от `kty`)**

#### **Для RSA (`kty: "RSA"`)**

- **`n`** — Модуль (modulus) в **Base64URL**.
- **`e`** — Публичный экспонент (public exponent), обычно `"AQAB"` (65537 в Base64URL).
- **`d`** — Приватный экспонент (private exponent) — **только для приватных ключей!**
- **`p`, `q`** — Простые множители (primes) — **только для приватных ключей!**
- **`dp`, `dq`, `qi`** — Дополнительные параметры для CRT (Chinese Remainder Theorem) — **только для приватных ключей!**

> **Важно:** В **публичном JWKS** (на эндпоинте `/jwks`) должны быть только `n`, `e`, `kty`, `kid`, `alg`, `use`. Приватные параметры (`d`, `p`, `q` и т.д.) **не expose!**




|Параметр|Описание|Формат|Требования к значению|Длина (в битах)|
|---|---|---|---|---|
|**`n`**|Модуль (modulus)|Base64URL|**Большое простое число** (произведение `p * q`)|**2048, 3072 или 4096 бит** (рекомендуется ≥2048)|
|**`e`**|Публичный экспонент|Base64URL|**Целое число**, обычно **65537** (0x10001)|16–32 бита (чаще всего 17 бит)|
|**`d`**|Приватный экспонент|Base64URL|**Целое число**, вычисляется из `n` и `φ(n)`|≈ длина `n`|
|**`p`**|Первый простой множитель|Base64URL|**Большое простое число**|≈ половина длины `n`|
|**`q`**|Второй простой множитель|Base64URL|**Большое простое число**|≈ половина длины `n`|
|**`dp`**|`d mod (p-1)`|Base64URL|**Целое число**|≈ длина `p`|
|**`dq`**|`d mod (q-1)`|Base64URL|**Целое число**|≈ длина `q`|
|**`qi`**|`q⁻¹ mod p`|Base64URL|**Целое число**|≈ длина `p`|


#### **Для EC (`kty: "EC"`)**

- **`crv`** — Имя кривой (например, `"P-256"`, `"P-384"`).
- **`x`, `y`** — Координаты публичной точки в **Base64URL**.
- **`d`** — Приватный ключ (private key) — **только для приватных ключей!**

#### **Для симметричных ключей (`kty: "oct"`)**

- **`k`** — Симметричный ключ в **Base64URL**.

---

### **6. `x5c` (X.509 Certificate Chain)**

- **Назначение:** Цепочка сертификатов X.509 (в **Base64**), связанных с ключом.
- **Использование:** Иногда используется для валидации ключей через сертификаты.
- **Формат хранения:** Массив строк (`["MII...", "MII..."]`).

---

### **7. `x5t` (X.509 Certificate SHA-1 Thumbprint)**

- **Назначение:** Отпечаток сертификата (SHA-1) в **Base64URL**.
- **Использование:** Для быстрой идентификации сертификата.
- **Формат хранения:** Строка.

---

### **8. `x5u` (X.509 URL)**

- **Назначение:** URL, по которому можно получить сертификат.
- **Формат хранения:** Строка-URL.

## **Кратко о хранении**

|Компонент|Формат хранения|Приватный?|Обязательный?|
|---|---|---|---|
|`kty`|Строка|Нет|Да|
|`kid`|Строка|Нет|Нет|
|`alg`|Строка|Нет|Нет|
|`use`|Строка|Нет|Нет|
|`n`|Base64URL|Нет|Да (для RSA)|
|`e`|Base64URL|Нет|Да (для RSA)|
|`d`|Base64URL|**Да**|Нет|
|`p`, `q`|Base64URL|**Да**|Нет|
|`x`, `y`|Base64URL|Нет|Да (для EC)|
|`k`|Base64URL|**Да**|Да (для oct)|
