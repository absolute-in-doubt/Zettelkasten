# @Transactional на методах service - Spring

## 🔎 Без `@Transactional` на сервисе

- Когда ты вызываешь метод репозитория (`userRepository.save(...)` или `findBy...`), Spring Data JPA сам открывает транзакцию **только на время выполнения этого метода**.

- После выхода из метода транзакция закрывается, `EntityManager` отсоединяет все сущности → они становятся **detached**.

- Если ты потом работаешь с ленивыми коллекциями (`user.getSessions().add(...)`), Hibernate не может подтянуть данные, потому что сессия уже закрыта → `LazyInitializationException`.


## ✅ С `@Transactional` на сервисе

- Аннотация `@Transactional` на методе сервиса говорит Spring: «Открой транзакцю при входе в метод и закрой её при выходе».

- Внутри метода создаётся **одна транзакция и один** `EntityManager`, который живёт всё время выполнения метода.

- Все вызовы репозиториев используют этот общий `EntityManager`.

- Сущности остаются **managed** до конца метода → можно безопасно работать с ленивыми коллекциями, добавлять новые объекты, каскадно сохранять и т.д.

- В конце метода транзакция коммитится (или откатывается при исключении).
