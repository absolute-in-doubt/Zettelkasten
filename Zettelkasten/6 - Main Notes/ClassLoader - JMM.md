# ClassLoader - JMM

Загрузчик классов является частью JRE, которая динамически загружает Java классы в JVM. Обычно классы загружаются только по запросу. Система исполнения в Java не должна знать о файлах и файловых системах благодаря загрузчику классов





### Иерархия загрузчиков

При запуске JVM, используются три загрузчика классов:

1. Bootstrap class loader (Загрузчик класса Bootstrap).

2. Platform (Extension) class loader (Загрузчик класса расширений). 
   Child of Bootstrap.

3. Application (System) class loader (Системный загрузчик классов).
   Child of Extensions loader.


**Загрузчик класса Bootstrap** загружает основные библиотеки Java, расположенные в папке `<JAVA_HOME>/jre/lib`. Этот загрузчик является частью ядра JVM, написан на нативном коде. ~={green}Не является экземпляром `java.lang.ClassLoader`.=~

**Загрузчик класса расширений Platform (Extension)** загружает код в каталоги расширений (`<JAVA_HOME>/jre/lib/ext`, или любой другой каталог, указанный системным свойством `java.ext.dirs`).

**Системный загрузчик** загружает код, найденный в `java.class.path`, который сопоставляется с переменной среды `CLASSPATH`. Это реализуется классом `sun.misc.Launcher$AppClassLoader`.


## Модель делегирования (parent‑first по умолчанию)

По умолчанию работает **~={green}модель делегирования parent‑first=~**:

- Когда некоторый `ClassLoader` должен загрузить класс, он:
    
    - Сначала проверяет, не загружен ли этот класс уже им самим.
        
    - Затем делегирует запрос _родителю_ (parent).
        
    - Если родитель не нашёл класс (бросил `ClassNotFoundException`), текущий загрузчик пытается загрузить его сам (из своих путей/источников).​
        
- Это обеспечивает:
    
    - **Уникальность**: класс, который смог загрузить родитель, не будет переопределён дочерним.
        
    - **Видимость**: дочерний загрузчик видит классы, загруженные родителем, но не наоборот.​
        

То есть по умолчанию схема действительно «снизу вверх делегируем, сверху вниз при необходимости пробуем сами», но инициатор запроса — текущий (обычно более «низкий») загрузчик, который сначала отдаёт шанс родителям.

### Пользовательские ClassLoaders

- Позволяет загружать классы из нестандартных мест. Даже получать по HTTP например;

- Изменять способ загрузки байт-кода (например, можно использовать зашифрованный байт-код класса Java);

- Модифицировать загруженный байт-код (например, для переплетения аспектов во время загрузки при использовании аспектно-ориентированного программирования);
