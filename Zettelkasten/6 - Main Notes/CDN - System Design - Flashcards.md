Q: Что такое CDN и для чего она нужна?
A: CDN - Content Delivery Network. Используется для уменьшения задержек в обслуживании пользователей из разных частей земного шара.
	
CDN — это не только про кэширование статического контента, но и более широкая инфраструктура для распределенной доставки данных. Оно может включать сети реплик или кластеры серверов для динамического контента тоже

Q: Когда стоит использовать CDN, а когда - исклбчительно Origin server?
A:    
**CDN применим для:**
- доставки статического контента - HTML, CSS, JS, video content
- Необходима низкая задержка для обработки запросов из разных частей земного шара.
- В приоритете снижение нагрузки на основной сервер
	
---
**Лучше использовать Origin Server для:**
- Возврата динамически изменяемого контента (часто изменяется или персонализирован под пользователя)
- Выполнение запросов, которое требует обработки up-to-date данных (типо менющихся цен на акции)
- Приложение требует сложную логику, котрая не может быть реплецирована или кэширована на CDN.
