# CQRS - Patterns

CQRS (Command Query Responsibility Segregation) — это архитектурный паттерн, разделяющий операции чтения данных (queries) и изменения данных (commands) на независимые модели. Он помогает оптимизировать системы с неравномерной нагрузкой, где чтение преобладает над записью.

## Основная идея

Паттерн предлагает использовать разные модели для команд (например, создание или обновление данных) и запросов (получение данных без изменений). Команды фокусируются на валидации и сохранении, не возвращая данные, а запросы — на быстрой выдаче информации через денормализованные views или отдельные БД.

Это решает проблемы традиционного CRUD, где одна модель перегружается, усложняя масштабирование и тестирование.

## Преимущества

- Масштабирование: чтение и запись можно распределять по разным сервисам или БД (SQL для записи, NoSQL для чтения).
    
- Оптимизация: разные модели данных — строгая для записи, упрощенная для чтения.​
    
- Простота: легче тестировать и сопровождать сложную бизнес-логику.​
    

## Когда применять

Идеален для микросервисов, высоконагруженных систем (e-commerce, соцсети) или DDD-проектов. Не стоит для простых CRUD-приложений — добавляет сложность.

## Реализация

Обычно сочетается с Event Sourcing: события из команд обновляют read-модель асинхронно. В коде — отдельные handler'ы для CommandHandler и QueryHandler.
