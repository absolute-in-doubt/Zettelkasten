# PCR - JMM


Регистр PC (Program Counter) — это специальный регистр процессора, который хранит адрес следующей команды, подлежащей выполнению. Он определяет, «где сейчас находится» процессор в последовательности исполнения программы.

**PC = Program Counter** — программный счётчик (часто также называют счётчиком команд или счётчиком адреса команд).


---
### PCR + JVM

Виртуальная машина Java может поддерживать много потоков исполнения одновременно. Каждый поток виртуальной машины Java имеет свой собственный регистр PC (program counter). В любой момент каждый поток виртуальной машины Java выполняет код одного метода, а именно текущий метод для этого потока. Если этот метод не является native, регистр pc содержит адрес инструкции виртуальной машины Java, выполняемой в настоящее время.


Коротко говоря: для одного потока, существует один PCR, который создается при запуске потока. PCR хранит адрес выполняемой сейчас инструкции JVM.


---
## Как он работает при последовательном исполнении

- На каждом шаге цикла выборки‑выполнения процессор:
    
    - считывает команду по адресу, указанному в PC;
        
    - после выборки автоматически увеличивает PC на размер инструкции (или фиксированное количество байт, в зависимости от архитектуры), чтобы он указывал на _следующую_ команду.​
        
- Таким образом, при линейном коде программа идёт «вперёд» просто за счёт автоинкремента PC.
