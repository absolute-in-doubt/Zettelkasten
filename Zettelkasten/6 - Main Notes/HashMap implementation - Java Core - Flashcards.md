Q: Сколько bucket-ов в HashMap изначально? По какому принципу (когда?) создаются новые buckets?
A:   
Default amount of buckets = 16. Используя конструкторы с параметрами можно задавать произвольное начальное количество корзин.
	
**HashMap в Java автоматически увеличивает количество bucket-ов при достижении порогового значения загрузки (threshold).** Это происходит **и** при превышении 75% заполнения таблицы (load factor = 0.75 по умолчанию), **и** независимо от преобразования содержимого bucket-ов в деревья.
	
| Событие       | Условие                  | Действие                |
| ------------- | ------------------------ | ----------------------- |
| **Resize**    | `size > capacity × 0.75` | Увеличение bucket-ов ×2 |
| **Treeify**   | Bucket > 8 элементов     | LinkedList → Tree       |
| **Untreeify** | Bucket < 6 элементов     | Tree → LinkedList       |
	
> [!note]
> - **size**: общее количество пар ключ-значение (13 в вашем примере). Метод `hashMap.size()` вернёт 13.
>    
>- **capacity**: размер внутреннего массива бакетов (16 по умолчанию)

Q: Какова сорость записи и чтения из HashMap?
A: Обычно - O(1) и то и то, но если содержимое bucket-ов перестраивается в red-black tree, то в худшем случае:
**Скорость чтения:**
- O(log n) 
**Скорость записи:**
- O(log n) + resize
