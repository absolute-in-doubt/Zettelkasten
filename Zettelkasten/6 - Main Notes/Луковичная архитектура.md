### Service

Знает всё про сущность, работает с ней и **только** с ней; но он не знает ничего более - в частности, он ничего не знает и не должен знать про то, с какими подключаемыми интерфейсами работает приложение и каким образом происходит подключение. Он ничего не знает и не должен знать про хранение данных, сетевые протоколы, UI и прочее. Только логика.

Логический слой целиком и полностью зависит от бизнес-сущности, и больше ни от чего не зависит.

![[Pasted image 20251107082821.png]]

> [!note] 
> Т.к. сервису нобходимо взаимодействовать с БД, но он не должен знать о реализации DAO и Repository -> используем интерфейс для DAO (или Repository):
> ```java
> public class UserServiceImpl implements UserService {
>
>private UserDAO userDao; //хранит ссылку на интерфейс, но о реализации не знает
>
> // реализация методов UserService
> } 
> ```
> 
> ```java
> public interface UserDAO {
> void persist(User user);
> void update(User user, int id);
> void remove(int id);
> void get(int id);
> }
> ```

### Слой DAO / Repository

Знает о том, с какой БД работает приложение, по какому протоколу происходит сетевое взаимодействие, какие дополнительные сервисы требуются для доступа к тем или иным данным. При этом, DAO ничего не знает о работе внешних систем - только о подключении к ним.

![[Pasted image 20251107083653.png]]

~={red}**Компоненты слоя DAO имеют собственные модели, позволяющие им работать с внешними интерфейсами. Таковыми являются DTO для контроллера и репозитарные сущности для репозитория.**=~

#### Как компоненты DAO взаимодействуют данными с бизнес-слоем?

- Сервис знает только про бизнес-сущность и оперирует только ей.

- DAO знает про бизнес-сущность и про свои DTO. В части взаимодействия с внешними интерфейсами DAO оперирует DTO. В части взаимодействия с бизнес-слоем DAO оперирует бизнес-сущностью.


Обмен данными в пределах слоя DAO:

![Обмен данными в пределах слоя DAO](https://habrastorage.org/r/w1560/getpro/habr/upload_files/f73/744/91e/f7374491e3ed698ca75303369231ba18.png "Обмен данными в пределах слоя DAO")

~={red}**Таким образом, мапинг из бизнес-сущности в DTO и обратно реализуется в слое DAO.**=~
