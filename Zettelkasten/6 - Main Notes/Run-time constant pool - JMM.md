# Run-time constant pool - JMM

A run-time constant pool существует для каждого класса или интерфейса в рантайме и представленно constant_pool таблицей в `.class` файле

Каждый run-time constant pool отделён от JVM's method area.

Для каждого загруженного класса или интерфейса создаётся свой runtime constant pool.

### Что тут хранится?

- Числовые литералы (int, long, float, double) и строковые литералы. Эти значения используются инструкциями байткода через индексы в пуле.

- ***~={green}Символьные=~*** ссылки:
	- на классы/интерфейсы (имя типа);
	
	- на поля (к какому классу относятся, имя и тип);
	
	- на методы и интерфейсные методы (класс-владелец, имя, сигнатура);
	
	- вспомогательные записи вроде NameAndType, Utf8 и др..


### Для чего нужен

Когда Java класс скомпилирован, все ссылки на переменные и методы хранятся в class constant pool как символические ссылки. Символические ссылки это логические ссылки, и они не указывают на адреса физической памяти. JVM реализация может выбрать когда произвести замену ссылок на адреса (resolve), это может произойти когда класс прошёл верификацию, после загрузки, так называемое статическое связывание, или же это может произойти при первом вызове (позднее) связывание.

> [!example] Note
> JVM нужно произвести замену символической ссылки на ссылку на класс или метод. Этот процесс происходит при первом использовании, а не при компиляции или загрузке в память. Если при выполненнии JVM выявляет ошибку (например класс на который ссылались не существует), то будет брошено исключение. Таким образом возможна ситуация, когда JVM запущена, но при выполнении кода возникли ошибки.

|                                             | Runtime constants pool                                                                                                                   | Method Area                                                                                                                            |
| ------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------- |
| Обычные поля примитивного типа              | Сами значения **не хранятся**; могут быть только константы, использованные в инициализации (как литералы)                                | Хранится описание поля (имя, тип, модификаторы); само значение — в каждом объекте в heap, а не в Method Area.                          |
| Обычные ссылочные поля                      | Не хранятся значения ссылок; есть только символьные ссылки `Fieldref` на поля (класс, имя, тип).                                         | Хранится описание поля; реальные ссылки на объекты лежат в экземплярах в heap, а не в Method Area.                                     |
| static primitive fields                     | В пуле может быть константа‑литерал для `static final`, если инициализируется константным выражением​.                                   | Есть слот в таблице статических полей класса, где хранится само примитивное значение в «сырых» битах​.                                 |
| static ссылочные поля                       | В пуле есть символьная ссылка `Fieldref` (класс, имя, тип) и, возможно, строковый/другой литерал для инициализации​.                     | В таблице статических полей хранится **сама Java‑ссылка** на объект в heap; описание поля — в метаданных класса​.                      |
| Кто получает хранимые значения (назначение) | Значения/записи читают интерпретатор и JIT: для `ldc` (литералы), а также для разрешения ссылок на классы/поля/методы по индексам пула​. | Значения статических полей читают/меняют инструкции `getstatic`/`putstatic` и код методов; метаданные используют JVM, JIT и рефлексия. |
