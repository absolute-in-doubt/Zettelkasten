# Event Sourcing - Patterns

Event Sourcing — это паттерн хранения состояния приложения как последовательности неизменяемых событий, а не текущего снимка данных. Вместо обновления записи в БД система фиксирует каждое изменение (например, "OrderCreated", "OrderPaid") в специальном хранилище событий (Event Store), откуда состояние можно восстановить путём воспроизведения всех событий в хронологическом порядке.

## Преимущества

- Полная история изменений для аудита, отладки и отката.
    
- Гибкость: события можно проецировать в разные read-модели (для CQRS).
    
- Высокая целостность данных и поддержка сложной бизнес-логики.
    

## Пример

Счёт с балансом: вместо хранения "balance: 100" хранятся события "Deposited 50", "Withdrawn 30", "Deposited 80". Текущее состояние = сумма всех событий.
