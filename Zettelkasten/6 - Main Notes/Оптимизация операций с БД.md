### Оптимизация запросов


![[Pasted image 20260215171245.png]]

Смотрим план, видим **[[Организация доступа к данным - PostgreSQL|Seq scan]]**:
1. Добавляем индекс (concurrently)
2. Видим **[[Организация доступа к данным - PostgreSQL|Index Scan/Index Only Scan]]** 

Индекс есть, но он говно (не используется):
- индекс на Boolean, Enum - плохо делят (2 - немного кучек данных)
- используются функции -> не используются индексы `LOWER(email) = "email@dfs.com`
- без индекса быстрее - если маленькая таблица


Индекс используется:

Признаки:
![[Pasted image 20260215171333.png]]

1. Планировщик думает одно, а по факту другое
- планировщик опирается на статистику, а autovacuum не справился - статистика не корректна.

1. Index Only Scan стал ходить в Heap
- visibility map устарела - AUTOVACUUM не почистил -> тюним его, чтоб чистил чаще

1. ..
- Пагинация в OFFSET... LIMIT (вычитывает все записи, но отправляет только часть) -> пагинация по Id (по индексу) -> вычитает по диапазону, не станет читать все
- index bloat, table bloat - раздутие - строчки не лежат рядом -> тюнинг 
- `fill_factor` - оставим место для обновлений -> сразу раздуваем, но в дальнейшем записи (новые версии) хранятся на тех же самых страницах.
- AUTOVACUUM, 
- `pg_repack`, `pg_compacttable`  - vacuum full без stop the world ``
