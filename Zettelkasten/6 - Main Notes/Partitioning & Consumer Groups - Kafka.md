# Partitioning - Kafka

Партиционирование - разделение топика на более мелкие независимые фрагменты - partitions.

![[Pasted image 20260201094207.png]]

Partition - содержит копию данных, помещаемых в Topic.
Каждый partition представляет собой лог, в котором сообщения хранятся в порядке их поступления. 


Partitioning **~={red}позволяет=~** Kafka **~={red}параллелизировать обработку данных=~**, что даёт возможность нескольким потребителям (consumers) одновременно читать данные из разных разделов.

Данные в каждом partition хранятся последовательно (append-only log) на диске. Это позволяет эффективно писать и читать сообщения с высокой пропускной способностью.


---
### Consumer groups


Consumer group = логическая группа потребителей, которые совместно читают данные из одного или нескольких топиков, деля между собой партиции. Это значит, что каждая партиция топика обрабатывается только одним потребителем внутри группы. То есть, в одной consumer group обычно находятся все экземпляры одного сервиса, которые выполняют одну и ту же задачу. Kafka отслеживает смещения (offsets) отдельно для каждой группы, чтобы гарантировать, что одно сообщение не будет прочитано двумя потребителями из одной группы.

![[Pasted image 20260201101057.png]]

> [!note]
>  У каждого Consumer внутри Consumer group - НЕ свой partition внутри topic. `__consumer_offsets` - один топик (реплицируемый), куда все Consumer Groups пишут свои offsets.
> 
> Однако offset у каждого Consumer свой. Т.е. если два Consumer из разных Consumer group читают один и тот же partition - каждый может читать в разном месте, и у каждого свой offset.

![[Pasted image 20260207115505.png]]

Коммит состоит из 4 частей:
- А - имя топика
- 0 - номер partition - Нет смысла писать имя Consumer-а, т.к. один и тот же partition читается одним и тем же Consumer, однако, если тот падает, Consumer меняется.
- X - имя Consumer Group
- 2 - количество offsets которое надо сдвинуть
