# Канальный vs Сетевой уровень OSI - Networks

Исторически и архитектурно **IP и MAC решают разные задачи**.


> **IP отвечает на вопрос "куда в мире (или среди подсетей) доставить пакет", а MAC — "кому прямо сейчас передать кадр по этому физическому сегменту сети".**

### Представь аналогию с почтой

- **IP-адрес** — это почтовый адрес: страна, город, улица, дом.
    
- **MAC-адрес** — это имя человека, которому почтальон вручает письмо в конкретном доме.
    

---

## Почему нельзя обойтись только IP?

Потому что Ethernet (и многие другие L2-технологии) изначально устроен так, что устройства в локальной сети обмениваются **кадрами**, а каждый Ethernet-кадр обязан содержать:

```text
Destination MAC
Source MAC
Payload (например, IP-пакет)
```

Коммутаторы Ethernet работают **только с MAC-адресами**. Они вообще не обязаны понимать IP.

Например, коммутатор видит:

```text
MAC AA:AA:AA -> порт 1
MAC BB:BB:BB -> порт 7
```

и отправляет кадр на нужный порт.

---

## Что происходит при отправке пакета соседу по локальной сети?

Допустим:

```text
PC1: 192.168.1.10
PC2: 192.168.1.20
```

Приложение хочет отправить пакет на `192.168.1.20`.

Но сетевой интерфейс не может сказать:

> «Передай это устройству с IP 192.168.1.20».

Он умеет только:

> «Передай Ethernet-кадр устройству с MAC XX:XX:XX».

Поэтому сначала используется протокол **ARP**:

```text
Кто имеет IP 192.168.1.20?
```

PC2 отвечает:

```text
192.168.1.20 -> MAC BB:BB:BB
```

После этого отправляется кадр:

```text
Dst MAC = BB:BB:BB
Src MAC = AA:AA:AA

Payload:
    IP src = 192.168.1.10
    IP dst = 192.168.1.20
```

---

## Почему MAC всё ещё нужен сегодня?

Потому что он позволяет:

1. **Коммутаторам работать очень быстро**, не анализируя IP-пакеты.
    
2. Делать канальный уровень независимым от сетевого уровня.
    
3. Использовать поверх Ethernet не только IP, но и другие протоколы (исторически это были, например, IPX, AppleTalk и др.).
    
4. Передавать кадры даже до получения IP-адреса (например, во время DHCP).
    

---

### Для собеседования

Можно ответить так:

> IP-адрес используется для логической адресации и маршрутизации между сетями, а MAC-адрес — для доставки кадров внутри локальной сети. Ethernet-коммутаторы работают на канальном уровне и пересылают кадры именно по MAC-адресам, поэтому перед отправкой пакета по IP устройство сначала определяет соответствующий MAC-адрес с помощью ARP.
