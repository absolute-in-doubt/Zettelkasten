# Zookeepeer vs KRaft - Kafka

В новых версиях Apache Kafka (на февраль 2026 года) ZooKeeper **больше не используется по умолчанию** и полностью удалён начиная с Kafka 4.0 (выпущена в конце 2025 года). Вместо него применяется встроенный режим **KRaft** (Kafka Raft Metadata), который управляет метаданными кластера самостоятельно через кворум контроллеров.

## Статус поддержки ZooKeeper

- **Kafka 3.5+ (2024–2025)**: ZooKeeper помечен как deprecated, с возможностью миграции на KRaft.
    
- **Kafka 4.0+ (2025–2026)**: Полное удаление поддержки ZooKeeper — кластеры запускаются только в режиме KRaft.
    
- Миграция: Автоматическая через этапы dual-write (запись в оба хранилища), после чего ZooKeeper можно отключить.​
    

## KRaft vs ZooKeeper

| Аспект           | ZooKeeper (legacy) ​      | KRaft (текущий)        |
| ---------------- | ------------------------- | ---------------------- |
| **Роль**         | Внешний сервис метаданных | Встроенный Raft-кворум |
| **Масштаб**      | Ограничен ZK-узлами       | До 1000+ брокеров      |
| **Эксплуатация** | Два кластера (Kafka + ZK) | Один кластер Kafka     |

KRaft упрощает деплой, повышает надёжность и устраняет зависимость от внешнего ZooKeeper.
