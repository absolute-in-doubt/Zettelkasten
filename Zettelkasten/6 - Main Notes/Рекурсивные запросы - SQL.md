# Рекурсивные запросы - SQL


`WITH RECURSIVE` позволяет описать временную таблицу (CTE), которая строится в несколько шагов: сначала базовый набор строк, затем многократное добавление новых строк по рекурсивному правилу. Такой запрос часто используют для деревьев `id/parent_id` или графов.

---

## Базовая структура запроса

Общий шаблон:

```sql
WITH RECURSIVE cte_name (col1, col2, ...) AS (
    -- 1. Анкор‑запрос (базовый уровень)
    SELECT ...
    FROM ...
    WHERE ...          -- стартовые строки

    UNION ALL          -- или UNION (обычно ALL)

    -- 2. Рекурсивная часть (ссылается на cte_name)
    SELECT ...
    FROM ...
    JOIN cte_name ON ...   -- связь с предыдущим уровнем
    WHERE ...              -- условие продолжения
)
-- 3. Основной запрос по результату CTE
SELECT *
FROM cte_name;
```

Ключевые элементы:

- Имя CTE (`cte_name`) и список столбцов, которыми вы будете оперировать.
    
- Анкор‑часть — один обычный `SELECT`, который задаёт начальное множество строк.
    
- Рекурсивная часть — `SELECT`, который ссылается на `cte_name` и добавляет новые строки на основании уже найденных.
    
- Между ними стоит `UNION ALL`: рекурсивная часть выполняется снова и снова, пока на очередной итерации не перестанет возвращать новые строки.
    

---

## Пример для дерева категорий

Структура:

```sql
categories (
    id        bigint primary key,
    name      text,
    parent_id bigint references categories(id)
);
```

Запрос, который берёт категорию `:root_id` и всех её потомков:

```sql
WITH RECURSIVE category_tree(id, name, parent_id, depth) AS (
    -- Анкор: корневая категория
    SELECT
        c.id,
        c.name,
        c.parent_id,
        0 AS depth
    FROM categories c
    WHERE c.id = :root_id

    UNION ALL

    -- Рекурсивная часть: дети уже отобранных категорий
    SELECT
        c.id,
        c.name,
        c.parent_id,
        ct.depth + 1 AS depth
    FROM categories c
    JOIN category_tree ct ON c.parent_id = ct.id
)
SELECT *
FROM category_tree
ORDER BY depth, id;
```

Что здесь важно:

- В анкоре выбирается стартовая категория.
    
- В рекурсивной части `JOIN category_tree` задаёт движение по дереву.
    
- Поле `depth` показывает уровень вложенности, удобно для сортировки и отладки.
    

---

## Типичные расширения

- **Отбор/фильтрация**: можно добавлять условия в рекурсивную часть (например, ограничить глубину `WHERE ct.depth < 5`).
    
- **Защита от циклов**: в CTE можно хранить дополнительную колонку `path` (список id) и не переходить к узлам, которые уже есть в пути.
    
- **Произвольный SELECT после CTE**: вместо `SELECT * FROM cte_name` можно делать агрегации, джойны и т.п. по результату рекурсии.
